# v1.4.0 流量配额冒烟测试

覆盖 Feature 013 (per-(user, client) 月度流量配额 + 历史聚合) 的端到端
人工冒烟流程。E2E 自动化在 `crates/portunus-e2e/tests/traffic_quotas.rs`,
本文档只在开发机 / staging 上手动跑一次,确认 UI、数据面、Prometheus
都一致。

## 准备

```sh
make clean              # 清掉 /tmp/portunus-dev 旧状态
make dev                # 启动后端 + Vite UI
```

记录终端打印的 `_superadmin` 临时密码,访问 http://localhost:5173 登录
并改密。

## 1. 客户端连接

Web UI → **Clients** → **Provision** → 名字 `edge-test`,下载 `bundle.json`。
另开终端启动客户端:

```sh
target/debug/portunus-client --bundle <下载路径>
```

**预期:** Clients 列表中 `edge-test` 状态变成 `Connected`(5 秒内)。

## 2. 用户 + 授权

1. Web UI → **Users** → **Create** → `alice`,普通用户。
2. 进入 alice 详情页 → 创建凭证(记下 token)。
3. 在配额表里给 alice 加一个 `edge-test` 入口,端口 9100,TCP。

## 3. 启动上游 echo

```sh
# 终端 A
ncat -lk 127.0.0.1 9200 -c 'cat'
# 或:socat TCP-LISTEN:9200,reuseaddr,fork EXEC:cat
```

## 4. 推送转发规则(以 alice 身份)

```sh
curl -X POST http://127.0.0.1:7080/v1/rules \
  -H "Authorization: Bearer <alice token>" \
  -H "Content-Type: application/json" \
  -d '{"client":"edge-test","listen_port":9100,"target_host":"127.0.0.1","target_port":9200,"protocol":"tcp"}'
```

**预期:** Rules 列表里规则状态 `Active`。

## 5. 配额 PUT(superadmin 身份)

```sh
curl -X PUT "http://127.0.0.1:7080/v1/users/alice/quotas/edge-test" \
  --cookie "<superadmin session cookie>" \
  -H "Content-Type: application/json" \
  -H "X-Portunus-CSRF: 1" \
  -d '{"monthly_bytes": 1048576}'      # 1 MiB
```

刷新 UserDetail:**Monthly quota** 列显示 `1.0 MiB`,**This period** 进
度条 0%。

## 6. 验证数据面计量

quota 同时计入站和回包,所以一次 echo 会按 `2 × payload` 扣预算。
推 256 KiB 让本期用量到 ~50%:

```sh
# 终端 B - 推 256 KiB:
head -c 262144 /dev/urandom | nc 127.0.0.1 9100 | wc -c
# 应该返回 262144(完整 echo 回来,扣掉 ~512 KiB 预算)
```

刷新 **UserDetail** → **Traffic**(Last 1 hour、bucket 1m):

- This period 进度条 ~50%(等下一次 StatsReport,周期 5s)。
- TrafficPanel 出现一根面积柱。

## 7. 触发硬关闭

```sh
# 推 2 MiB,会被截断:
head -c 2097152 /dev/urandom | nc 127.0.0.1 9100 | wc -c
# 返回字节数 ≪ 2097152(剩余预算被一次 echo 吃光,写端半关后立即 EOF)
```

**预期:**

- UserDetail 顶部出现红色 ExhaustedBanner:
  *"Monthly quota exhausted / Forwarding paused for edge-test"*。
- This period 进度条变红色,显示 `Exhausted`。
- `GET /v1/users/alice/quotas/edge-test/status` 返回
  `"exhausted": true`,`"exhausted_at": <unix>`。

## 8. Prometheus 指标

本地 `make dev` 下 `http://127.0.0.1:7080/metrics` 走 SPA fallback,
真正的 Prometheus 文本暴露在带鉴权的 operator `/v1/metrics`:

```sh
curl -s --cookie "<superadmin cookie>" \
  http://127.0.0.1:7080/v1/metrics | grep traffic_quota
```

应该看到(`_exhausted_total` 是单调 counter,本轮耗尽过几次就是几):

```
portunus_traffic_quota_bytes_used{user="alice",client="edge-test"} 1048576
portunus_traffic_quota_bytes_limit{user="alice",client="edge-test"} 1048576
portunus_traffic_quota_exhausted{user="alice",client="edge-test"} 1
portunus_traffic_quota_exhausted_total{user="alice",client="edge-test"} 1
```

## 9. 清零本期用量

在 ExhaustedBanner 中点击 **Clear usage**(仅 superadmin 可见),或:

```sh
curl -X PATCH "http://127.0.0.1:7080/v1/users/alice/quotas/edge-test" \
  --cookie "<superadmin cookie>" -H "X-Portunus-CSRF: 1" \
  -H "Content-Type: application/json" \
  -d '{"clear_period_usage": true}'
```

**预期:**

- `/status` 立即返回 `used=0`,`exhausted: false`,横幅消失,进度条
  归零;新流量立刻放行(再推 64 KiB 可以完整 echo 回来)。
- Prometheus `portunus_traffic_quota_exhausted` gauge 与
  `_bytes_used` 在 status 翻转后**不**立即更新,要等到下一次 client
  `StatsReport`(默认 5s)抵达。`_exhausted_total` 是单调 counter,
  不会回退。

## 10. UDP 路径(可选)

把规则改成 UDP(`protocol: "udp"`)加一个新的小配额,用 `nc -u` 验证:
同样有 `is_exhausted` 短路与逐数据报 consume。

## 11. 重连重放顺序(可选,验证 C5)

杀掉 `portunus-client` 进程再重启:

```sh
grep "traffic_quota.applied_set\|rule.installed" <client stderr>
```

**预期:** `traffic_quota.applied_set` 事件出现在 `rule.installed`
**之前**(配额先于规则到达)。

## 12. 周期边界(可选,只能模拟)

无法在生产路径直接验证,需要等到 `billing_anchor` 真到下一个月 28~31 日
时观察 `_period_resets_total` 自增 + `current_period_started_at` 推进。
或写一次性脚本把数据库里的 `billing_anchor` 调到 31 天前,等下一次 60s
rollover tick。

## 2026-05-14 冒烟执行记录

环境:macOS 本机,`make clean && make dev`,Chrome + Computer Use 手工验证,
`target/debug/portunus-client --bundle /tmp/edge-test.bundle.json`,本地 Ruby TCP
echo 上游 `127.0.0.1:9200`。

通过:

- Web UI 登录成功,UserDetail 显示 `alice`/`edge-test` 配额行:
  `1.0 MiB`,本期 `0 B (0%)`,状态在线。
- 未连接 client 时 PUT quota 返回
  `422 quota_unsupported_by_client`,client 连接后 PUT 成功且 status 立即反映
  `monthly_bytes=1048576`,`exhausted=false`。
- `edge-test` 在 5 秒内进入 connected,alice 下发 TCP 规则后返回
  `status=Active`。
- 256 KiB echo 完整返回,等待一个 StatsReport 周期后 status/UI 显示
  `512.0 KiB (50%)`,TrafficChart 出现样本。
- 从 50% 用量继续推 2 MiB 时只回读 `16384` B,status 变为
  `exhausted=true`,`exhausted_at` 有 unix 秒;UserDetail 顶部红色横幅出现,
  进度条显示 `1.0 MiB (100%) 已耗尽`。
- ClientDetail 顶部也渲染 exhausted banner:`edge-test 已暂停转发`。
- PATCH `clear_period_usage` 后 status 立即变为
  `used=0`,`remaining=1048576`,`exhausted=false`;随后 64 KiB echo 完整返回,
  下一个 StatsReport 周期后 UI 显示 `128.0 KiB (13%)`。
- `CHANGELOG.md`、英文/中文 traffic quota runbook、troubleshooting 文档中都有
  v1.4.0 quota 内容。

发现偏差:

- 本机 dev server 的 `http://127.0.0.1:7080/metrics` 返回 SPA HTML;实际可用的
  operator HTTP 镜像是 authenticated `GET /v1/metrics`。通过 `/v1/metrics`
  看到 `portunus_traffic_quota_bytes_used`,
  `portunus_traffic_quota_bytes_limit`,
  `portunus_traffic_quota_exhausted`,
  `portunus_traffic_quota_exhausted_total`。
- 文档第 6 步写“推 512 KiB 后约 50%”,但当前实现按入站+出站共同扣 quota。
  512 KiB echo 完整返回后 status 直接达到
  `used=1048576`,`exhausted=true`;要观察 50% 需要推 256 KiB。
- PATCH clear 后 status API 立即为 false,但 Prometheus gauge 未立即归零;需要下一次
  client StatsReport 后 `/v1/metrics` 才更新到
  `bytes_used=131072`,`exhausted=0`。放行新流量本身是立即生效的。
- `portunus_traffic_quota_exhausted_total` 是单调 counter,本轮多次触发耗尽后为
  `4`,不应期望每次清零后回到 `1`。
