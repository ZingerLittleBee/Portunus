# v1.4.0 流量配额冒烟测试

覆盖 Feature 013 (per-(user, client) 月度流量配额 + 历史聚合) 的端到端
人工冒烟流程。E2E 自动化在 `crates/portunus-e2e/tests/traffic_quotas.rs`,
本文档只在开发机 / staging 上手动跑一次,确认 UI、数据面、Prometheus
都一致。

## 准备

```sh
make clean              # 清掉 /tmp/portunus-dev 旧状态
make dev                # 启动后端 + Vite UI
```

记录终端打印的 `_superadmin` 临时密码,访问 http://localhost:5173 登录
并改密。

## 1. 客户端连接

Web UI → **Clients** → **Provision** → 名字 `edge-test`,下载 `bundle.json`。
另开终端启动客户端:

```sh
target/debug/portunus-client --bundle <下载路径>
```

**预期:** Clients 列表中 `edge-test` 状态变成 `Connected`(5 秒内)。

## 2. 用户 + 授权

1. Web UI → **Users** → **Create** → `alice`,普通用户。
2. 进入 alice 详情页 → 创建凭证(记下 token)。
3. 在配额表里给 alice 加一个 `edge-test` 入口,端口 9100,TCP。

## 3. 启动上游 echo

```sh
# 终端 A
ncat -lk 127.0.0.1 9200 -c 'cat'
# 或:socat TCP-LISTEN:9200,reuseaddr,fork EXEC:cat
```

## 4. 推送转发规则(以 alice 身份)

```sh
curl -X POST http://127.0.0.1:7080/v1/rules \
  -H "Authorization: Bearer <alice token>" \
  -H "Content-Type: application/json" \
  -d '{"client":"edge-test","listen_port":9100,"target_host":"127.0.0.1","target_port":9200,"protocol":"tcp"}'
```

**预期:** Rules 列表里规则状态 `Active`。

## 5. 配额 PUT(superadmin 身份)

```sh
curl -X PUT "http://127.0.0.1:7080/v1/users/alice/quotas/edge-test" \
  --cookie "<superadmin session cookie>" \
  -H "Content-Type: application/json" \
  -H "X-Portunus-CSRF: 1" \
  -d '{"monthly_bytes": 1048576}'      # 1 MiB
```

刷新 UserDetail:**Monthly quota** 列显示 `1.0 MiB`,**This period** 进
度条 0%。

## 6. 验证数据面计量

```sh
# 终端 B - 推 512 KiB:
head -c 524288 /dev/urandom | nc 127.0.0.1 9100 | wc -c
# 应该返回 524288(完整 echo 回来)
```

刷新 **UserDetail** → **Traffic**(Last 1 hour、bucket 1m):

- This period 进度条 ~50%。
- TrafficPanel 出现一根面积柱(StatsReport 周期 5s,可能要等)。

## 7. 触发硬关闭

```sh
# 推 2 MiB,会被截断:
head -c 2097152 /dev/urandom | nc 127.0.0.1 9100 | wc -c
# 应该 < 2097152(配额耗尽,写端被半关)
```

**预期:**

- UserDetail 顶部出现红色 ExhaustedBanner:
  *"Monthly quota exhausted / Forwarding paused for edge-test"*。
- This period 进度条变红色,显示 `Exhausted`。
- `GET /v1/users/alice/quotas/edge-test/status` 返回
  `"exhausted": true`,`"exhausted_at": <unix>`。

## 8. Prometheus 指标

```sh
curl -s http://127.0.0.1:7080/metrics | grep traffic_quota
```

应该看到:

```
portunus_traffic_quota_bytes_used{user="alice",client="edge-test"} 1048576
portunus_traffic_quota_bytes_limit{user="alice",client="edge-test"} 1048576
portunus_traffic_quota_exhausted{user="alice",client="edge-test"} 1
portunus_traffic_quota_exhausted_total{user="alice",client="edge-test"} 1
```

## 9. 清零本期用量

在 ExhaustedBanner 中点击 **Clear usage**(仅 superadmin 可见),或:

```sh
curl -X PATCH "http://127.0.0.1:7080/v1/users/alice/quotas/edge-test" \
  --cookie "<superadmin cookie>" -H "X-Portunus-CSRF: 1" \
  -H "Content-Type: application/json" \
  -d '{"clear_period_usage": true}'
```

**预期:**

- 横幅消失,进度条回到 0%,`exhausted: false`。
- 再推 512 KiB 又能完整 echo 回来。
- `portunus_traffic_quota_exhausted` 回到 0。

## 10. UDP 路径(可选)

把规则改成 UDP(`protocol: "udp"`)加一个新的小配额,用 `nc -u` 验证:
同样有 `is_exhausted` 短路与逐数据报 consume。

## 11. 重连重放顺序(可选,验证 C5)

杀掉 `portunus-client` 进程再重启:

```sh
grep "traffic_quota.applied_set\|rule.installed" <client stderr>
```

**预期:** `traffic_quota.applied_set` 事件出现在 `rule.installed`
**之前**(配额先于规则到达)。

## 12. 周期边界(可选,只能模拟)

无法在生产路径直接验证,需要等到 `billing_anchor` 真到下一个月 28~31 日
时观察 `_period_resets_total` 自增 + `current_period_started_at` 推进。
或写一次性脚本把数据库里的 `billing_anchor` 调到 31 天前,等下一次 60s
rollover tick。

## 关键检查清单

- [ ] 配额未设时数据流量正常,但 `traffic_samples_1m` 表里也能查到样本
      (B2 总是 upsert)。
- [ ] Quota 设置后 status 立即反映。
- [ ] 数据面在 ~ 一个 buffer 内 hard-kill,不会无限超额。
- [ ] PATCH `clear_period_usage` 立即放行新流量。
- [ ] PUT 提高限额时,数据面自动恢复。
- [ ] ExhaustedBanner 在 UserDetail 顶部 / ClientDetail 顶部都能渲染。
- [ ] TrafficChart 至少在 5s StatsReport 周期后显示样本。
- [ ] CHANGELOG / 文档显示在 docs site (`pnpm dev` in `docs/`)。
