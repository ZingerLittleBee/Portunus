`portunus-standalone` 是一个完全由 TOML 文件驱动的独立 TCP/UDP 转发器。它与
`portunus-client` 共享同一套数据面代码，但无需运行 `portunus-server` ——
适用于单机部署、边缘设备以及不需要控制面服务器的场景。

## 快速上手

```sh
cargo build --release -p portunus-standalone
# →  target/release/portunus-standalone
```

创建一个最简配置 `portunus.toml`：

```toml
[[rule]]
name = "ssh"
protocol = "tcp"
listen_port = 2222
target = "10.0.0.5:22"
```

运行：

```sh
./target/release/portunus-standalone --config portunus.toml
```

仅验证配置，不启动服务：

```sh
./target/release/portunus-standalone --check --config portunus.toml
# 配置合法时打印 "ok" 并以 0 退出，配置有误时以 2 退出
```

## 配置文件查找顺序

省略 `--config` 时，按以下顺序查找：

1. `$PORTUNUS_STANDALONE_CONFIG`（环境变量）
2. `./portunus.toml`（当前工作目录）

如果所有路径均不存在，打印详细错误并以 2 退出。

## 配置文件结构

```toml
# [global] 节 — 所有字段均可选
[global]
label              = "my-forwarder"   # 人类可读标签，仅供参考
log_level          = "info"           # tracing EnvFilter 指令（默认："info"）
log_format         = "json"           # "json"（默认）或 "pretty"
shutdown_drain_secs = 30              # SIGTERM 后的优雅关闭等待时间（默认：30 秒）

# [defaults] 节 — 规则未指定字段时的默认值
[defaults]
protocol          = "tcp"    # 可选；规则仍可单独覆盖
udp_max_flows     = 1024     # 每条规则的 UDP 流表上限（默认：1024）
udp_flow_idle_secs = 60      # 空闲流过期时间，单位秒（默认：60）
prefer_ipv6       = false    # 地址族偏好（默认：false，即优先 IPv4）

# [[rule]] — 至少需要一条
[[rule]]
name          = "ssh-tunnel"          # 唯一名称，用于日志输出
protocol      = "tcp"                 # "tcp" 或 "udp"
listen_port   = 2222                  # 单端口  }  二选一
# listen_ports = "8000-8009"          # 端口范围 }
target        = "10.0.0.5:22"        # 单目标   }  二选一
# targets = [...]                     # 多目标   }
prefer_ipv6   = false                 # 每条规则的覆盖项（可选）
udp_max_flows = 1024                  # UDP 规则的覆盖项（可选）
udp_flow_idle_secs = 60              # UDP 规则的覆盖项（可选）
```

### 端口范围

使用 `listen_ports` 配合 `"lo-hi"` 字符串，将连续监听端口窗口映射到目标端的同偏移窗口：

```toml
[[rule]]
name         = "web-range"
protocol     = "tcp"
listen_ports = "8000-8009"
target       = "10.0.0.10:8000-8009"
```

监听端口范围与目标端口范围的宽度必须相同。

### 多目标故障转移

使用 `targets` 列表形式实现按优先级排列的故障转移：

```toml
[[rule]]
name        = "ha-https"
protocol    = "tcp"
listen_port = 8443
targets = [
  { host = "primary.internal",   port = 443, priority = 0,  proxy_protocol = "v2" },
  { host = "secondary.internal", port = 443, priority = 10, proxy_protocol = "v2" },
]
```

`priority` 值越小优先级越高。转发器会优先尝试优先级最高的健康目标；连接失败时按优先级顺序回退。

### PROXY 协议

在任意目标（单目标或 `targets = [...]` 列表内）设置 `proxy_protocol = "v1"` 或
`"v2"`，可在上游连接建立前发送 PROXY 协议头，使后端获得原始客户端地址。

## CLI 参数

| 参数 | 默认值 | 说明 |
| --- | --- | --- |
| `--config <PATH>` | （自动查找） | TOML 配置文件路径 |
| `--check` | false | 仅验证配置并退出（0 = 合法，2 = 有误） |
| `--log-level <LEVEL>` | 读取配置 | 覆盖日志级别 |
| `--log-format <FMT>` | 读取配置 | 覆盖日志格式（`json` 或 `pretty`） |

## 信号处理

| 信号 | 行为 |
| --- | --- |
| `SIGTERM` / `SIGINT` | 优雅关闭 — 活跃连接在 `shutdown_drain_secs` 内排空后进程以 0 退出。 |
| `SIGHUP` | 当前版本为空操作（配置热重载尚未支持）。 |

## 可观测性

`portunus-standalone` 默认将日志以 **JSON** 格式输出到 **stderr**
（开发时可设置 `log_format = "pretty"` 以获得可读输出）。每条规则在启动时
发出结构化事件，周期性报告器每隔 30 秒记录各规则的字节数和连接数统计。

本二进制不内嵌 HTTP 服务、Prometheus 端点或 gRPC 流——这些能力属于
`portunus-server`。请使用日志聚合工具（如 `journald`、sidecar 日志采集器）
收集 JSON 日志。

## 完整配置示例

```toml
[global]
log_level = "debug"

[defaults]
udp_max_flows = 4096

[[rule]]
name        = "ssh-tunnel"
protocol    = "tcp"
listen_port = 2222
target      = "10.0.0.5:22"

[[rule]]
name         = "web-range"
protocol     = "tcp"
listen_ports = "8000-8009"
target       = "10.0.0.10:8000-8009"

[[rule]]
name        = "ha-https"
protocol    = "tcp"
listen_port = 8443
targets = [
  { host = "primary.internal",   port = 443, priority = 0,  proxy_protocol = "v2" },
  { host = "secondary.internal", port = 443, priority = 10, proxy_protocol = "v2" },
]

[[rule]]
name             = "game-udp"
protocol         = "udp"
listen_port      = 27015
target           = "10.0.0.20:27015"
udp_flow_idle_secs = 120
```

## 与 portunus-client 的差异

| 功能 | `portunus-client` | `portunus-standalone` |
| --- | --- | --- |
| 控制面 | 连接 `portunus-server` 的 gRPC 流 | 无——仅 TOML |
| 动态规则更新 | 是（服务端推送） | 否——需重启 |
| RBAC / 多用户 | 是 | 否 |
| 限速 / QoS | 是 | 否 |
| Prometheus 指标 | 是（本地 HTTP） | 否 |
| 审计日志 | 经由服务端 | 结构化日志输出到 stderr |
| SNI 路由 | 是 | 否 |
| TCP splice 快速路径（Linux） | 是 | 是（共享同一数据面代码） |
| PROXY 协议输出 | 是 | 是 |

## 生产环境部署

四种部署方式的参考模板均位于
[`crates/portunus-standalone/contrib/`](https://github.com/ZingerLittleBee/Portunus/tree/main/crates/portunus-standalone/contrib)。

### Installer（二进制 + systemd，推荐）

生命周期安装器（`scripts/install.sh`）把 `standalone` 当作与 `client`
和 `server` 平级的 role：

```sh
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/scripts/install.sh | bash -s -- standalone
```

安装器会下载 release tarball、将二进制装到 `/usr/local/bin/portunus-standalone`、
创建 `portunus` 系统用户/组、把加固版 unit 文件写到
`/etc/systemd/system/portunus-standalone.service`（源文件位于
[`contrib/portunus-standalone.service`](https://github.com/ZingerLittleBee/Portunus/blob/main/crates/portunus-standalone/contrib/portunus-standalone.service)），
并在 `/etc/portunus/standalone.toml` 放一份起步配置。编辑该文件后执行
`systemctl start portunus-standalone` 即可启动。

卸载：`bash install.sh uninstall standalone`（清理 unit 与二进制；加
`--purge` 一并删除 `/etc/portunus`）。

### Docker / docker-compose

参考
[`Dockerfile`](https://github.com/ZingerLittleBee/Portunus/blob/main/crates/portunus-standalone/contrib/Dockerfile)
是一个两阶段构建，运行时镜像基于 distroless。
[`docker-compose.yml`](https://github.com/ZingerLittleBee/Portunus/blob/main/crates/portunus-standalone/contrib/docker-compose.yml)
默认使用 host 网络，端口范围转发因此无需逐端口 mapping：

```sh
cd crates/portunus-standalone/contrib
docker compose up -d
docker compose logs -f
```

目前未发布 GHCR 镜像 —— 请本地构建或推送到自有 registry。

### Kubernetes

[`contrib/k8s/`](https://github.com/ZingerLittleBee/Portunus/tree/main/crates/portunus-standalone/contrib/k8s)
中的 manifest 运行一个单副本 `Deployment`，启用 `hostNetwork: true`
（任意端口转发的硬性要求）：

```sh
kubectl apply -f crates/portunus-standalone/contrib/k8s/configmap.yaml
kubectl apply -f crates/portunus-standalone/contrib/k8s/deployment.yaml
```

Deployment 与 DaemonSet 的取舍以及 PodSecurityPolicy 注意事项见
[`contrib/k8s/README.md`](https://github.com/ZingerLittleBee/Portunus/blob/main/crates/portunus-standalone/contrib/k8s/README.md)。

### 加固版 systemd unit

[`contrib/portunus-standalone.service`](https://github.com/ZingerLittleBee/Portunus/blob/main/crates/portunus-standalone/contrib/portunus-standalone.service)
设置了 `LimitNOFILE=65535`、以 `User=portunus` 运行、新增
`AmbientCapabilities=CAP_NET_BIND_SERVICE` 以便绑定特权端口，并启用
`ProtectSystem=strict` / `PrivateTmp=true` / `NoNewPrivileges=true`。
二进制不支持 SIGHUP 热加载 —— 修改 TOML 后请执行
`systemctl restart portunus-standalone`。
