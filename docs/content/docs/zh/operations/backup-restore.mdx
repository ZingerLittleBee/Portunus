v0.8 起，server 端所有持久化制品都存在 `<data-dir>/state.db` 里。Portunus
提供配套的 backup / restore / reset CLI 子命令。

## 备份

```sh
portunus-server backup --out ~/portunus-backup-$(date +%F).db
```

走 SQLite Online Backup API（`rusqlite::backup::Backup`）。备份 **兼顾 WAL**，
无须暂停写入即可保持一致 —— 在线服务期间运行也安全。

它会拒绝覆写已存在的目标 —— 请选一个唯一路径（按日期命名是惯例）。若 `--out`
指向一个已存在的目录，备份会写成该目录下的 `portunus-state-<RFC3339>.db`。

校验：

```sh
sqlite3 ~/portunus-backup-2026-05-08.db \
    'SELECT version FROM schema_migrations ORDER BY version DESC LIMIT 1;'
```

## Cron 调度

```sh
# /etc/cron.daily/portunus-backup
#!/bin/sh
set -eu
DEST=/var/backups/portunus/$(date +%F).db
mkdir -p "$(dirname "$DEST")"
sudo -u portunus portunus-server \
  --data-dir /var/lib/portunus \
  backup --out "$DEST"
find /var/backups/portunus -name '*.db' -mtime +30 -delete
```

## 恢复

```sh
# 1. 停掉正在运行的 server。
sudo systemctl stop portunus-server

# 2. 清空本地 SQLite 数据库。
portunus-server reset --confirm

# 3. 恢复。
portunus-server restore --in /tmp/portunus-backup-2026-05-08.db

# 4. 启动。
sudo systemctl start portunus-server
```

`restore`：

- 拷贝前先校验 SQLite header 魔数。
- 不带 `--force` 时拒绝覆写非空 data-dir。带 `--force` 时会先删除
  `state.db` 及 `-wal` / `-shm` / `.lock` 边车，再拷入备份。
- 备份的 schema 比当前二进制旧时，自动跑 schema 迁移。
- 迁移或打开失败时，回滚已写一半的 `state.db`（及其边车），避免下次
  启动读到损坏文件。
- 备份的 schema **比当前二进制还新** 时，**拒绝**并以 `78` 退出，事件
  `startup.schema_version_too_new`。要么保留旧备份，要么用对应版本的二进制
  来恢复。

## Reset

```sh
# 干跑（不带 --confirm）打印将要删除的内容。
portunus-server reset

# 真正清除。
portunus-server reset --confirm
```

清除 `state.db` 与边车 `state.db-wal`、`state.db-shm`、`state.db.lock`。

如果路径的前 16 字节不是 SQLite header，reset CLI 会**拒绝**动作 —— 防止
`--data-dir` 写错。它**不会**清空同目录中的 TLS 材料（`server.crt` /
`server.key`）或可选的 `server.toml`。对空 data-dir 执行 `reset` 是 no-op，
退出码 0。

## 跨主机迁移

v0.5–v0.7 的 `identity.json` 是 mode-0600 的原子写 JSON，可以放心 `rsync`。
`state.db` 则**不行** —— 跨主机迁移请走 `backup` / `restore` 流程。

## 常见坑

- **`--data-dir` 落在 NFS / tmpfs / ramfs** 时，启动期会以
  `event=startup.unsupported_filesystem` 拒绝。请用本地可写文件系统。
- **两个 `portunus-server serve` 同时打开同一 data-dir** 时，第二个会以
  `event=startup.store_in_use`（退出 75）退出。clustering 不在当前版本范围。
- **`reset --confirm` 不会清空 server.crt / server.key / `server.toml`** ——
  只清 SQLite state。如需一起清掉，请单独删除。
