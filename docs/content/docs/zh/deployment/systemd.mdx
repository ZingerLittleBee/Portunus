`Portunus` 在 `deploy/systemd/` 目录下提供了拿来就能用的加固 unit。两份
unit 都设置了 `User=`、`ProtectSystem=full`、`CapabilityBoundingSet=`、
以及与负载相称的 `LimitNOFILE` 覆写。

## 一键安装

```sh
cd deploy/systemd
sudo ./install.sh
```

脚本会：

- 创建服务用户（`portunus-server`、`portunus-client`）。
- 按 FHS 铺好目录：
  - `/var/lib/portunus/` —— `state.db`、自动生成的 TLS 材料、可选 `server.toml`（mode 0750）。
  - `/etc/portunus/` —— client bundle 目录。
- 安装 `portunus-server.service` 和 `portunus-client.service`。

## server.service

```ini
[Unit]
Description=Portunus server
After=network.target

[Service]
User=portunus-server
Group=portunus-server
StateDirectory=portunus
StateDirectoryMode=0700
ExecStart=/usr/local/bin/portunus-server serve \
    --data-dir /var/lib/portunus
Restart=on-failure
RestartSec=2
TimeoutStopSec=35
ProtectSystem=full
ProtectHome=true
NoNewPrivileges=true
CapabilityBoundingSet=CAP_NET_BIND_SERVICE
AmbientCapabilities=CAP_NET_BIND_SERVICE
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
```

`StateDirectory=portunus` 是 systemd 原生写法：systemd 会用正确的属主
自动创建 `/var/lib/portunus`，并把 `$STATE_DIRECTORY=/var/lib/portunus`
注入进程。

`TimeoutStopSec=35` 在配置的 drain（`shutdown_drain_timeout_secs = 30`）
之上多留 5 秒余量。

## client.service

```ini
[Unit]
Description=Portunus client
After=network.target

[Service]
User=portunus-client
Group=portunus-client
ExecStart=/usr/local/bin/portunus-client \
    --bundle /etc/portunus/client.bundle.json
Restart=on-failure
RestartSec=2
TimeoutStopSec=35
ProtectSystem=full
ProtectHome=true
NoNewPrivileges=true
CapabilityBoundingSet=CAP_NET_BIND_SERVICE
AmbientCapabilities=CAP_NET_BIND_SERVICE
LimitNOFILE=65535

[Install]
WantedBy=multi-user.target
```

## 首次启动流程

安装后：

```sh
# 1. 可选：覆写默认值
sudo cp deploy/server.toml.example /var/lib/portunus/server.toml
sudo nano /var/lib/portunus/server.toml

# 2. 启用并启动 server
sudo systemctl enable --now portunus-server
sudo systemctl status portunus-server

# 3. 读取 serve 打印的 setup token
sudo journalctl -u portunus-server -n 100 --no-pager \
  | grep 'Portunus onboarding setup token'

# 4. 通过 SSH tunnel 或本机浏览器打开 http://127.0.0.1:7080/，
#    粘贴 setup token，创建第一个 superadmin 密码。

# 5. 在 Web UI Clients 页面创建首个 Client，或用 CLI 打印一次性接入命令。
portunus-server enroll-client edge-01

# 6. 在边缘主机执行输出的 `portunus-client enroll 'portunus://...'`。
#    它会写入 /etc/portunus/client.bundle.json。

# 7. 在边缘主机上启用 client unit
sudo systemctl enable --now portunus-client
```

setup token 30 分钟后过期，并且在 onboarding 完成前每次 server 启动都会轮换。

## 密码恢复

如果最后一个 `superadmin` 忘记 Web 密码，先停服务，在本地重置现有账户，再启动
服务。这里要使用真实的超管用户 ID：`bootstrap-superadmin` 创建的是
`_superadmin`，Web onboarding 创建的是初始化时选择的 ID：

```sh
sudo systemctl stop portunus-server
sudo -u portunus-server portunus-server \
  --data-dir /var/lib/portunus \
  reset-password admin --temporary
sudo systemctl start portunus-server
```

该命令只打印一次临时密码，并要求登录后修改密码。它会直接打开 SQLite store，
所以不要在 `portunus-server.service` 运行时执行。

## 加固备注

- **小于 1024 的端口** 需要 `CAP_NET_BIND_SERVICE`（unit 已包含）。
  转发器只绑规则配置过的端口，能力面被严格收敛。
- **`--data-dir` 落在 NFS / tmpfs / ramfs** 启动期会以
  `event=startup.unsupported_filesystem` 拒绝。请改本地文件系统。
- **两个 server 进程同时打开同一 data-dir** 第二个会以
  `event=startup.store_in_use`（退出 75）退出。clustering 不在范围。

## 备份 cron

```sh
# /etc/cron.daily/portunus-backup
#!/bin/sh
set -eu
DEST=/var/backups/portunus/$(date +%F).db
mkdir -p "$(dirname "$DEST")"
sudo -u portunus-server portunus-server \
  --data-dir /var/lib/portunus \
  backup --out "$DEST"
find /var/backups/portunus -name '*.db' -mtime +30 -delete
```

在线备份走 SQLite Online Backup API —— 不需要静默写入，可在 server
正常服务期间安全运行。
