Railway 可以把 `portunus-server` 跑成一个服务：Web UI 和 operator HTTP API
走 Railway 的公开 HTTP 域名，`portunus-client` 连接用的 gRPC control plane
则通过 Railway TCP Proxy 暴露到应用端口 `7443`。

## 模板结构

Railway 构建使用 [`deploy/railway/Dockerfile`](https://github.com/ZingerLittleBee/Portunus/blob/main/deploy/railway/Dockerfile)。
它先构建 React Web UI，把 `webui/dist/` 嵌入 `portunus-server`，再用一个精简的
Debian runtime image 启动 server。

启动时，[`deploy/railway/start-server.sh`](https://github.com/ZingerLittleBee/Portunus/blob/main/deploy/railway/start-server.sh)
会写入最小的 `/var/lib/portunus/server.toml`：

```toml
control_listen = "0.0.0.0:7443"
operator_http_listen = "0.0.0.0:${PORT}"
metrics_listen = "127.0.0.1:7081"
# 仅在解析出 public origin 时才追加（见下文）：
operator_http_public_origin = "https://${RAILWAY_PUBLIC_DOMAIN}"
```

`$PORT` 是 Railway 分配给 HTTP 服务的端口。control-plane listener 固定在
`7443`，方便 Railway TCP Proxy 单独暴露。前三行总会写入；
`operator_http_public_origin` 这一行只有在解析出 public origin 时才追加
—— 优先取 `PORTUNUS_OPERATOR_HTTP_PUBLIC_ORIGIN`（若已设置），否则取 Railway
的 `RAILWAY_PUBLIC_DOMAIN`。两者都没设时，这一行会整行省略。绑定自定义域名时，
设置 `PORTUNUS_OPERATOR_HTTP_PUBLIC_ORIGIN` 即可。

## 必需的 Railway 资源

在 Railway 项目里配置：

| 资源 | 值 |
| --- | --- |
| Service build | Dockerfile，路径 `deploy/railway/Dockerfile` |
| Volume mount | `/var/lib/portunus` |
| Public HTTP networking | 启用，使用 Railway 生成的域名即可 |
| TCP Proxy | application port `7443` |
| Service variable | `PORTUNUS_ADVERTISED_ENDPOINT=<tcp-proxy-host>:<tcp-proxy-port>` |

Volume 不是可选项。它保存 `state.db`、自动生成的 TLS 材料和 server config。没有
volume，每次 redeploy 都会得到一台全新的 server，operator 状态也会丢失。

Railway 的 volume 以 `root` 挂载；Railway 镜像也特意以 `root` 运行，这样挂载的
`/var/lib/portunus` 默认可写，无须额外设置 service variable。

创建 TCP Proxy 后再设置 `PORTUNUS_ADVERTISED_ENDPOINT`。Railway 会在 TCP Proxy
设置里显示生成的 proxy domain 和 proxy port。请填这个精确的 `host:port`；在设置
该变量之前生成的接入命令会写入错误的 endpoint，需要重新创建。

## 第一次初始化

部署服务后打开 Railway HTTP 域名，Web UI 位于 `/`。

如果 store 是空的，server 日志会打印 setup token：

```sh
Portunus onboarding setup token: <token>
```

把 token 粘进 Web UI，然后创建第一个 superadmin。token 30 分钟后过期；只要
onboarding 还没完成，每次重启都会轮换。

## Client endpoint

给 application port `7443` 创建 TCP Proxy 后，把 `PORTUNUS_ADVERTISED_ENDPOINT`
设为 proxy 的 `host:port` 并重新部署。之后从 Web UI 的 **Clients** 页面接入 client；
接入命令会让 `portunus-client` 连接 TCP Proxy endpoint。用 Docker（参见
[Client 配置](/zh/docs/configuration/client#docker)）或宿主机安装程序
（[`install.sh`](/zh/docs/configuration/client#安装二进制)）安装并运行边缘客户端。

## 注意事项

- Railway HTTP 域名只用于 Web UI 和 `/v1/*` operator API。
- `portunus-client` 必须连接 TCP Proxy endpoint，不要连 HTTP 域名。
- Prometheus metrics 仍然只监听 `127.0.0.1:7081`，Railway 默认不会暴露。
- 只有在你故意改变 TCP Proxy application port 时，才需要设置
  `PORTUNUS_CONTROL_LISTEN`。默认值是 `0.0.0.0:7443`。
- 使用自定义域名时，设置
  `PORTUNUS_OPERATOR_HTTP_PUBLIC_ORIGIN=https://your.custom.domain`。
