`forward-rs` 是一个用 Rust 编写的高性能端口转发服务。它由一个中心化的
**控制面 Server** 和一个或多个 **边缘 Client** 组成：边缘 Client 在远端
主机上绑定监听端口，并将 TCP / UDP 流量转发到指定的上游目标。

操作员可以从 Server 端通过 CLI、HTTP API 或 Web UI 推送规则；每个边缘 Client
通过一条 TLS 指纹固定 (TLS-pinned) 的双向 gRPC 流接收规则。每条规则的字节、
连接、数据报指标每 5 秒上报一次，可通过 Prometheus 与操作员 API 查看。

## 为什么选择 forward-rs？

- **单二进制**，零运行时依赖。内置 SQLite、嵌入式 Web UI、内置 TLS —— 拷贝
  二进制即可运行。
- 控制面采用 **TLS + Bearer Token** 认证。Client 通过预先嵌入到下发包中的
  SHA-256 证书指纹 (pinning) 校验 Server 身份。
- **多租户 RBAC**。可将每个用户限定到指定 Client、端口范围与协议；规则的
  归属关系由 Server 端强制执行。
- **L4 字节直通**。即使按 SNI 路由，也绝不解密、终结或重新加密 TLS。
- **运维友好**。配额支持热更新、连接优雅 drain、审计日志、Prometheus
  指标、结构化 JSON 日志、备份 / 恢复工具一应俱全。
