# Portunus

Portunus 用来转发远端机器上的 TCP / UDP 端口。单机时，它可以读取一个 TOML 文件
直接运行；机器多了以后，可以用 Server 集中下发规则，并管理用户权限、限速、指标和
审计记录。业务流量只经过转发机器，不经过 Server，也不会被解密或改写。

只有一台主机？[standalone 转发器](/zh/docs/configuration/standalone)
读取本地 TOML，并复用同一套转发代码，不需要 Server 或注册步骤。

## 性能

在测试机上，未限速的 TCP 转发一路到 20 Gbit/s 都能跟上 `iptables`。当前性能报告
基于 v1.6.1 实测，包含测试机信息、命令、原始数字和注意事项。

| 场景 | 结论 |
| --- | --- |
| 100 Mbit/s - 20 Gbit/s | 在测试机上打满目标速率。 |
| Standalone 与 Client | 共用同一套转发代码，当前实测看不出吞吐差异。 |
| 带限速的规则 | 保留同样的指标、计数器和审计行为。 |

## 它提供什么

| 领域 | 能力 |
| --- | --- |
| 转发 | TCP / UDP 规则、端口范围、DNS 域名目标、多目标故障切换。 |
| 路由 | 按 HTTPS 域名分流但不解密 TLS，可选 PROXY protocol。 |
| 管理 | CLI、HTTP API、内置 Web UI，以及向已连接 Client 实时下发规则更新。 |
| 权限 | 按用户、Client、协议与端口范围授权，规则归属由 Server 强制执行。 |
| 限速 / 限额 | 按规则或用户设置带宽、新连接速率、并发连接上限。 |
| 运维 | Prometheus 指标、结构化日志、审计记录、内置 SQLite、备份与恢复。 |

## 适合什么场景

当你需要跨机器管理转发规则、给用户限定可用范围、查看流量，或按用户设置限额时，
`Portunus` 比一条静态的本机转发规则更合适。如果你只需要单机上的一条固定转发，
并且唯一目标就是极限吞吐，那么内核转发更简单。如果你想在单机上使用 Portunus 的
端口范围、多目标故障切换、PROXY protocol 等能力，但不需要 Server，就用
[standalone 转发器](/zh/docs/configuration/standalone)：同一套转发代码，
完全由 TOML 文件驱动。
