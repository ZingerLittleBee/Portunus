`portunus-client` 是边缘端二进制。它拨号 bundle 指定的 server 地址，
按 SHA-256 指纹验证证书，并通过双向 gRPC 流接收规则下发。

## 用法

```sh
portunus-client [--bundle <PATH>] [--shutdown-drain-timeout-secs <N>]
```

## Bundle 解析

省略 `--bundle` 时（v0.8 起），按以下顺序查找：

1. `$PORTUNUS_CLIENT_BUNDLE`
2. `$XDG_CONFIG_HOME/portunus/client.bundle.json`
3. `$HOME/.config/portunus/client.bundle.json`
4. `./client.bundle.json`

全部失败时退出 `1` 并把尝试过的全部路径打印出来。

## 标志

| 标志 | 默认 | 用途 |
| --- | --- | --- |
| `--bundle <PATH>` | （解析链） | 显式 bundle 路径 |
| `--shutdown-drain-timeout-secs <N>` | 30 | SIGTERM/SIGINT 时的 drain 预算 |

`portunus-client --help` 给出当前权威的标志一览。

## 环境变量

| 变量 | 用途 |
| --- | --- |
| `PORTUNUS_CLIENT_BUNDLE` | bundle 路径（解析链中优先级最高） |
| `RUST_LOG` | `tracing-subscriber` env-filter |

## 生命周期

1. 读取 bundle。
2. 解析 `server_endpoint`。
3. TLS 拨号；按 `server_cert_sha256` 固定 SHA-256。
4. 携带 Bearer 发送 Hello；接收 Welcome 与服务端可调参数。
5. 打开双向 `RuleUpdate` ↔ `RuleStatus` 流。
6. 对每条 `RuleUpdate`：
   - 分配 listener（TCP `bind` 或 UDP `bind`）。
   - 激活转发循环。
   - 回复 `Active`（或 `Failed(<reason>)`）。
7. 每 5 秒推送一次 `StatsReport`。

## Shutdown drain

`SIGTERM` / `SIGINT` 触发 drain：

- listener 立即停止 accept。
- 已转发的连接最长 drain `--shutdown-drain-timeout-secs`（默认 30 s）。
- 内核回收剩余 socket。

systemd unit 的 `TimeoutStopSec=` 应与之对齐，避免 drain 中途 `SIGKILL`。

## 日志

默认 JSON 行：

```json
{"event":"process.start"}
{"event":"control.connecting","endpoint":"…:7443"}
{"event":"control.tls_pinned","fingerprint_sha256":"f5e7c2a1..."}
{"event":"control.connected"}
{"event":"rule.received","rule_id":1,"listen_port":18080}
{"event":"rule.activated","rule_id":1,"listen_port":18080}
```

固定指纹不匹配：

```json
{"event":"control.tls_pinned_mismatch","expected":"…","got":"…"}
```

…然后非零退出。
