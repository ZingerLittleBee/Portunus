`forward-server` 从 `--config-dir` 指定目录中的 `server.toml` 读取配置。
仅使用默认值的最小配置也是合法的：下面每一项都可省略。

## 监听器

```toml
# gRPC 控制面 listener —— Client 拨号此地址。
# 跨主机部署绑 0.0.0.0；或绑到具体网卡（如 "10.0.0.5:7443"）。
control_listen = "0.0.0.0:7443"

# 操作员 HTTP API + Web UI。默认必须保留 loopback。
# 每条 /v1/* 请求都需要 Authorization: Bearer <token>，
# loopback 是纵深防御，不是唯一闸门。
operator_http_listen = "127.0.0.1:7080"

# Prometheus /metrics 端点。同样的 loopback 约束。
metrics_listen = "127.0.0.1:7081"
```

## TLS 证书

```toml
# 自签证书首次启动时若不存在会自动生成。
# 默认 SAN 仅含 hostname + "localhost" + 127.0.0.1 ——
# 路由到外网的部署应该换成由运维管理、SAN 含真实地址的证书。
# server.key 必须是 mode 0600（loader 拒绝更宽松的权限）。
tls_cert_path = "/var/lib/forward/server.crt"
tls_key_path  = "/var/lib/forward/server.key"
```

## 持久化

```toml
# Client 认证用的 Bearer Token 存储。
# 首次 `provision-client` 时创建。Mode 0600。
# Token 是 256-bit 随机；务必备份。
token_store_path = "/var/lib/forward/tokens.json"

# 操作员侧身份存储（用户 / 凭据 / Grant）。
# Mode 0600。原子写 JSON；可在多机间 rsync 做 HA。
operator_store_path = "/var/lib/forward/identity.json"
```

v0.8 起大部分状态搬入 `<data-dir>/state.db`（SQLite）。详见
[SQLite 存储](/zh/docs/features/sqlite-storage)。

## 引导捷径

```toml
# 操作员超管的一次性引导捷径（FR-006）。
# 当身份存储里还没有任何超管时，Server 会用此 Token 自动签出
# 保留账号 `_legacy` 超管。引导后保留此行（一旦有超管就被忽略）
# 或删除均可。
operator_token = "<43-char URL-safe-base64 token>"
```

生成 Token：`forward-server gen-token`。

或：`forward-server bootstrap-superadmin --name ops` 一次性打印新 Bearer。

## Drain 与日志

```toml
# SIGINT/SIGTERM 时已转发连接的优雅 drain 上限。
# 超时后内核会回收剩余 socket。
# 与 systemd unit 的 TimeoutStopSec 对齐，避免 drain 中途被 SIGKILL。
shutdown_drain_timeout_secs = 30

# "json"（默认 —— 结构化）或 "compact"（人类可读）。
log_format = "json"
```

## 端口范围

```toml
# 一条端口范围规则可跨的端口数上限。
# 每个端口在 Client 上消耗一个 TCP listener（一个 fd）。
# 默认 1024。调高之前请先调高 systemd unit 的 LimitNOFILE。
range_rule_max_ports = 1024
```

## UDP 参数

```toml
# 流空闲多久后回收上游 socket。
# 范围 30..=300。默认 60。
udp_flow_idle_secs = 60

# 每条规则同时存活的 UDP 流上限。
# 范围 1..=65535。默认 1024。
# 持续抖动超出后，超额流会让 forward_rule_flows_dropped_overflow_total 累加，
# 已存在的流不会被驱逐。
udp_max_flows_per_rule = 1024
```

## DNS 解析器

当前发布将 spec 固定的默认值硬编码在 `ResolverConfig`（Client 端），
尚未对外暴露：

| 参数 | 值 |
| --- | --- |
| `cache_floor` | 5 s |
| `cache_ceiling` | 5 min |
| `stale_while_error_grace` | 30 s |
| `attempt_timeout` | 3 s |
| `negative_cache_retry` | 3 s |
| `max_concurrent_resolves` | 64 |

未来可能在 `[resolver]` 子表暴露上述项。

## CLI 覆写

监听地址也可以从 CLI 覆写：

```sh
forward-server --config-dir ./srv \
  --control-listen 0.0.0.0:7443 \
  --operator-http-listen 127.0.0.1:7080 \
  --metrics-listen 127.0.0.1:7081 \
  serve
```

CLI 优先于 `server.toml`。

## 示例：production-ready server.toml

```toml
control_listen        = "0.0.0.0:7443"
operator_http_listen  = "127.0.0.1:7080"
metrics_listen        = "127.0.0.1:7081"

tls_cert_path         = "/etc/portunus/server.crt"
tls_key_path          = "/etc/portunus/server.key"

token_store_path      = "/var/lib/portunus/tokens.json"
operator_store_path   = "/var/lib/portunus/identity.json"

shutdown_drain_timeout_secs = 30
log_format = "json"

range_rule_max_ports   = 1024
udp_flow_idle_secs     = 60
udp_max_flows_per_rule = 1024
```
