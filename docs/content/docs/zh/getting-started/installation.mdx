`Portunus` 有两个运行角色：

- `portunus-server` 运行在控制面主机上。Operator 连接它，它保存状态，并把规则
  推送给已连接的 Client。
- `portunus-client` 运行在每台边缘主机上。它连接回 Server，绑定转发端口并搬运流量。

请把对应组件安装到承担该角色的主机上。真实部署通常是一台 Server 主机，加一台或
多台 Client 主机。

## 前置条件

- **curl** 用于一行安装脚本。
- **Docker / Podman** 用于推荐的容器路径。
- **Rust 1.88+ 稳定版** 仅在从源码编译时需要。`protoc` 已通过
  `prost-build` 内联，无需额外安装。
- 真实部署需要 **两台可互通的主机**；本地试用可用一台主机两个终端。
- **Linux 或 macOS**。Linux 是生产支持目标。

## 快速安装 Docker

在一台全新的 Linux 主机上，可以直接使用 Docker 官方的 convenience script：

```sh
sudo curl -fsSL https://get.docker.com | sh
```

## 安装 Server

最快的路径是一行安装脚本：

```sh
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/scripts/install.sh | sh -s -- server
```

脚本会检测操作系统和架构，下载对应的 release 二进制，校验其 SHA-256，
并将 `portunus-server` 安装到 `/usr/local/bin`。详见
[install.sh 标志](/zh/docs/configuration/client#安装二进制)；标志表（`--version`、`--bin-dir`、`--systemd`）对两个角色均适用。

Docker Compose 是推荐的生产部署路径。

### 在 Server 主机上复制粘贴安装

1. 创建并进入工作目录：

```sh
mkdir -p ~/portunus-server && cd ~/portunus-server
```

2. 写入 `docker-compose.yml`：

```sh
cat > docker-compose.yml <<'EOF'
services:
  server:
    image: ghcr.io/zingerlittlebee/portunus-server:latest
    container_name: portunus-server
    ports:
      - "7443:7443"
      - "127.0.0.1:7080:7080"
    volumes:
      - portunus-data:/var/lib/portunus
    command:
      - --data-dir
      - /var/lib/portunus
      - serve
      - --operator-http-listen
      - 0.0.0.0:7080
    restart: unless-stopped

volumes:
  portunus-data:
    name: portunus-data
EOF
```

3. 启动服务并验证：

```sh
docker compose up -d && docker compose logs -f -t
```

这已经足够完成默认 Server 安装。自动生成的 TLS 材料和 SQLite 状态都放在
`portunus-data` 卷里。

Web UI 在 Server 主机的 `http://127.0.0.1:7080/`。Client 控制面 listener
仍然是 `0.0.0.0:7443`。

### Compose 文件参考

```yaml
services:
  server:
    image: ghcr.io/zingerlittlebee/portunus-server:latest
    container_name: portunus-server
    ports:
      - "7443:7443"
      - "127.0.0.1:7080:7080"
    volumes:
      - portunus-data:/var/lib/portunus
    command:
      - --data-dir
      - /var/lib/portunus
      - serve
      - --operator-http-listen
      - 0.0.0.0:7080
    restart: unless-stopped

volumes:
  portunus-data:
    name: portunus-data
```

示例默认使用 `:latest`；如果需要完全可复现的生产部署，请固定到具体版本 tag，
例如 `:vX.Y.Z`。

如果需要非默认监听地址或其他覆写，再把可选 `server.toml` 放到
`<data-dir>/server.toml`，例如
`./server.toml:/var/lib/portunus/server.toml:ro`。

操作员命令、Client Compose、备份迁移和生产注意事项见
[Docker 部署](/zh/docs/deployment/docker)。

如果从源码编译：

```sh
git clone https://github.com/ZingerLittleBee/Portunus.git
cd Portunus
cargo build --release -p portunus-server
sudo install -m 0755 target/release/portunus-server /usr/local/bin/
```

如需手动下载二进制归档，请参见下方 [Release 二进制](#release-二进制)。

在 Server 主机上验证：

```sh
portunus-server --version
```

## 安装 Client

最快的路径是一行安装脚本：

```sh
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/scripts/install.sh | sh -s -- client
```

生产环境推荐以容器方式运行 Client——参见下方 Docker 小节。

### 接入并启动

向操作员申请一条接入命令——可通过 Web UI **Clients** 页面；也可在宿主机二进制
部署的服务端执行 `portunus-server enroll-client <name>`；或者对于 Docker 部署的
服务端，执行 `docker compose run --rm operator enroll-client <name>`（参见
[操作员命令](/zh/docs/deployment/docker#操作员命令)）。然后在边缘主机上：

```sh
portunus-client enroll 'portunus://...' --out ./client.bundle.json
portunus-client --bundle ./client.bundle.json
```

参见 [Client 配置 — 接入](/zh/docs/configuration/client#接入enroll)，
了解 bundle 解析顺序、`0600` 权限和默认路径。

### Docker

以宿主用户身份运行一次性接入和长驻容器，确保 `0600` bundle 由同一身份写入与读取：

```sh
# 一次性：把接入命令兑换为宿主文件。
docker run --rm --user "$(id -u):$(id -g)" -v "$PWD:/work" \
  ghcr.io/zingerlittlebee/portunus-client \
  enroll 'portunus://...' --out /work/client.bundle.json

# 长驻转发容器。
docker run -d --name portunus-client --network host \
  --user "$(id -u):$(id -g)" \
  -v "$PWD/client.bundle.json:/etc/portunus/client.bundle.json:ro" \
  ghcr.io/zingerlittlebee/portunus-client
```

Compose 等价写法（`compose.client.yaml`），在 bundle 已存在的情况下：

```yaml
services:
  client:
    image: ghcr.io/zingerlittlebee/portunus-client:latest
    container_name: portunus-client
    network_mode: host
    user: "${HOST_UID}:${HOST_GID}"
    volumes:
      - ./client.bundle.json:/etc/portunus/client.bundle.json:ro
    restart: unless-stopped
```

用以下命令启动：

```sh
HOST_UID=$(id -u) HOST_GID=$(id -g) docker compose -f compose.client.yaml up -d
```

Client 推荐使用 `--network host`，因为转发监听端口是在后续 operator 推送规则时
动态创建的。如果不用 host networking，需要显式发布每个转发监听端口。

参见 [Client 配置](/zh/docs/configuration/client#docker) 了解为何两条命令都要以
宿主用户身份运行，以及 [Docker 部署](/zh/docs/deployment/docker) 了解生产注意事项。

如果从源码编译：

```sh
git clone https://github.com/ZingerLittleBee/Portunus.git
cd Portunus
cargo build --release -p portunus-client
sudo install -m 0755 target/release/portunus-client /usr/local/bin/
```

如需手动下载二进制归档，请参见下方 [Release 二进制](#release-二进制)。

在 Client 主机上验证：

```sh
portunus-client --version
```

## Release 二进制

推荐的安装方式是 `install.sh`——它会自动解析最新版本，下载正确归档，并校验 SHA-256。

如需手动下载，请访问 GitHub
[releases 页面](https://github.com/ZingerLittleBee/Portunus/releases)，
选择适合你平台的归档。归档命名格式为
`portunus-<version>-<target>.tar.gz`，支持以下四个目标平台：

- `x86_64-unknown-linux-gnu`
- `aarch64-unknown-linux-gnu`
- `x86_64-apple-darwin`
- `aarch64-apple-darwin`

每次 release 同时发布对应的 `portunus-<version>-checksums.txt`。

解压并安装：

```sh
# pick the asset for your platform from the releases page, then:
tar -xzf portunus-<version>-<target>.tar.gz
sudo install -m 0755 portunus-<version>-<target>/portunus-server /usr/local/bin/
sudo install -m 0755 portunus-<version>-<target>/portunus-client /usr/local/bin/
```

## systemd

一步完成二进制安装和加固 systemd unit 的安装：

```sh
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/scripts/install.sh | sudo sh -s -- server --systemd
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/scripts/install.sh | sudo sh -s -- client --systemd
```

这会创建服务用户和目录，并安装加固 unit（仅 Linux）。如果使用源码 checkout 安装，
改用 `cd deploy/systemd && sudo ./install.sh server client`（二进制必须已在
`/usr/local/bin`）。

详见 [systemd 部署](/zh/docs/deployment/systemd) 中的 unit 文件解读与加固说明。
