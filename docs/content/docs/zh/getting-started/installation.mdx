`Portunus` 有两个运行角色：

- `portunus-server` 运行在控制面主机上。Operator 连接它，它保存状态，并把规则
  推送给已连接的 Client。
- `portunus-client` 运行在每台边缘主机上。它连接回 Server，绑定转发端口并搬运流量。

请把对应组件安装到承担该角色的主机上。真实部署通常是一台 Server 主机，加一台或
多台 Client 主机。

## 前置条件

- 推荐路径需要 **Docker / Podman**。
- **Rust 1.88+ 稳定版** 仅在从源码编译时需要。`protoc` 已通过
  `prost-build` 内联，无需额外安装。
- 真实部署需要 **两台可互通的主机**；本地试用可用一台主机两个终端。
- **Linux 或 macOS**。Linux 是生产支持目标。

## 快速安装 Docker

在一台全新的 Linux 主机上，可以直接使用 Docker 官方的 convenience script：

```sh
sudo curl -fsSL https://get.docker.com | sh
```

## 安装 Server

Server 推荐使用 Docker Compose 安装。

### 在 Server 主机上直接复制执行

1. 创建并进入工作目录：

```sh
mkdir -p ~/portunus-server && cd ~/portunus-server
```

2. 写入 `docker-compose.yml`：

```sh
cat > docker-compose.yml <<'EOF'
services:
  server:
    image: ghcr.io/zingerlittlebee/portunus-server:latest
    container_name: portunus-server
    network_mode: host
    volumes:
      - portunus-data:/var/lib/portunus
    restart: unless-stopped

volumes:
  portunus-data:
    name: portunus-data
EOF
```

3. 启动服务并验证：

```sh
docker compose up -d && docker compose logs -f -t
```

这已经足够完成默认 Server 安装。自动生成的 TLS 材料和 SQLite 状态都放在
`portunus-data` 卷里。

使用 host networking 后，Web UI 在 Server 主机的
`http://127.0.0.1:7080/`。Client 控制面 listener 仍然是 `0.0.0.0:7443`。

### Compose 文件参考

```yaml
services:
  server:
    image: ghcr.io/zingerlittlebee/portunus-server:latest
    container_name: portunus-server
    network_mode: host
    volumes:
      - portunus-data:/var/lib/portunus
    restart: unless-stopped

volumes:
  portunus-data:
    name: portunus-data
```
示例默认使用 `:latest`；如果需要完全可复现的生产部署，请固定到具体版本 tag，
例如 `:1.0.0`。

如果需要非默认监听地址或其他覆写，再把可选 `server.toml` 放到
`<data-dir>/server.toml`，例如 `./server.toml:/var/lib/portunus/server.toml:ro`。

操作员命令、Client Compose、备份迁移和生产注意事项见
[Docker 部署](/zh/docs/deployment/docker)。

如果使用宿主机二进制，下载 Server 主机对应的 release 归档，解压后只安装
`portunus-server`：

```sh
tar -xzf portunus-1.0.0-x86_64-unknown-linux-gnu.tar.gz
sudo install -m 0755 portunus-1.0.0-x86_64-unknown-linux-gnu/portunus-server /usr/local/bin/
```

如果从源码编译：

```sh
git clone https://github.com/ZingerLittleBee/Portunus.git
cd Portunus
cargo build --release -p portunus-server
sudo install -m 0755 target/release/portunus-server /usr/local/bin/
```

在 Server 主机上验证：

```sh
portunus-server --version
```

## 安装 Client

Client 也推荐使用 Docker Compose 安装。

### 在边缘主机上直接复制执行

先通过 Server 侧 operator 流程生成 client bundle，并把 bundle 文件拷贝到边缘
主机。下面假设 bundle 文件名是 `edge-01.bundle.json`。

1. 创建并进入工作目录：

```sh
mkdir -p ~/portunus-client && cd ~/portunus-client
```

2. 把 `edge-01.bundle.json` 放到这个目录中，然后写入
   `docker-compose.yml`：

```sh
cat > docker-compose.yml <<'EOF'
services:
  client:
    image: ghcr.io/zingerlittlebee/portunus-client:latest
    container_name: portunus-client
    network_mode: host
    volumes:
      - ./edge-01.bundle.json:/etc/portunus/client.bundle.json:ro
    restart: unless-stopped
EOF
```

3. 启动 Client 并验证：

```sh
docker compose up -d && docker compose logs -f -t
```

### Compose 文件参考

```yaml
services:
  client:
    image: ghcr.io/zingerlittlebee/portunus-client:latest
    container_name: portunus-client
    network_mode: host
    volumes:
      - ./edge-01.bundle.json:/etc/portunus/client.bundle.json:ro
    restart: unless-stopped
```

Client 推荐使用 `--network host`，因为转发监听端口是在后续 operator 推送规则时
动态创建的。如果不用 host networking，需要显式发布每个转发监听端口。

如果使用宿主机二进制，在边缘主机上从同格式 release 归档里只安装
`portunus-client`：

```sh
tar -xzf portunus-1.0.0-x86_64-unknown-linux-gnu.tar.gz
sudo install -m 0755 portunus-1.0.0-x86_64-unknown-linux-gnu/portunus-client /usr/local/bin/
```

如果从源码编译：

```sh
git clone https://github.com/ZingerLittleBee/Portunus.git
cd Portunus
cargo build --release -p portunus-client
sudo install -m 0755 target/release/portunus-client /usr/local/bin/
```

在 Client 主机上验证：

```sh
portunus-client --version
```

## systemd

```sh
cd deploy/systemd
sudo ./install.sh
# 创建：
#   /var/lib/portunus/    state.db + 自动生成的 TLS 材料 + 可选 server.toml
#   /etc/portunus/        client bundle 目录
#   portunus-server / portunus-client 系统用户
```

unit 安装器假设 `portunus-server` 与 `portunus-client` 已经在宿主机上，
通常来自上面的 release 归档。

详见 [systemd 部署](/zh/docs/deployment/systemd) 中的 unit 文件解读与加固说明。
