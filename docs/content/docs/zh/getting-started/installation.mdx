`Portunus` 提供两种部署模式，安装前先选定其一：

- **Server + Client（控制面）。** [`portunus-server`](#安装-server) 运行在控制面
  主机上——Operator 连接它，它保存状态，并把规则推送给已连接的 Client；
  [`portunus-client`](#安装-client) 运行在每台边缘主机上，连接回 Server、绑定转发
  端口并搬运流量。
- **Standalone（无控制面）。** [`portunus-standalone`](#安装-standalone) 是一个
  单独的二进制，所有规则都从本地 TOML 文件读取——没有 Server、没有 enrollment、
  没有 gRPC。它复用与 Client 完全相同的数据面代码，并自带一个终端实时流量统计界面
  （`portunus-standalone stats`）。

### 该安装哪一种？

| | [**Standalone**](#安装-standalone) | [**Server + Client**](#安装-server) |
| --- | :---: | :---: |
| 规则来源 | 本地 TOML 文件 | Server 实时推送 |
| 适用场景 | 单机 / 边缘盒子 | 多主机、集中管控 |
| 需要运维/加固/备份控制面 Server | 不需要 | 需要 |
| 热更新规则（无需重启） | ✗ | ✓ |
| 浏览器 Web UI、RBAC、审计日志 | ✗ | ✓ |
| 限速 / QoS、按 owner 配额 | ✗ | ✓ |
| TLS-SNI 路由 | ✗ | ✓ |
| Prometheus 指标 | ✗ | ✓ |
| 实时流量面板 | ✓ 终端 TUI（`portunus-standalone stats`） | ✓ Web UI + Prometheus |
| TCP/UDP、端口范围、多目标故障转移、PROXY protocol、DNS 目标 | ✓ | ✓ |

Server + Client 部署通常是一台 Server 主机加一台或多台 Client 主机；Standalone
部署就是一台主机加一个配置文件。本地试用一台主机两个终端即可。

## 前置条件

- **curl** 用于一行安装脚本。
- **Docker / Podman** 用于推荐的容器路径。
- **Rust 1.88+ 稳定版** 仅在从源码编译时需要。`protoc` 已通过
  `prost-build` 内联，无需额外安装。
- 真实部署需要 **两台可互通的主机**；本地试用可用一台主机两个终端。
- **Linux 或 macOS**。Linux 是生产支持目标。

## 快速安装 Docker

在一台全新的 Linux 主机上，可以直接使用 Docker 官方的 convenience script：

```sh
sudo curl -fsSL https://get.docker.com | sh
```

## 安装 Server

最快的路径是一行安装脚本：

```sh
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/scripts/install.sh | bash -s -- server
```

脚本会检测操作系统和架构，下载对应的 release 二进制，校验其 SHA-256，
并将 `portunus-server` 安装到 `/usr/local/bin`。详见
[install.sh 标志](/zh/docs/configuration/client#安装二进制)；标志表（`--version`、`--bin-dir`、`--systemd`）对两个角色均适用。

Docker Compose 是推荐的生产部署路径。

### 在 Server 主机上复制粘贴安装

1. 创建并进入工作目录：

```sh
mkdir -p ~/portunus-server && cd ~/portunus-server
```

2. 写入 `docker-compose.yml`：

```sh
cat > docker-compose.yml <<'EOF'
services:
  server:
    image: ghcr.io/zingerlittlebee/portunus-server:latest
    container_name: portunus-server
    ports:
      - "7443:7443"
      - "127.0.0.1:7080:7080"
    volumes:
      - portunus-data:/var/lib/portunus
    command:
      - --data-dir
      - /var/lib/portunus
      - serve
      - --operator-http-listen
      - 0.0.0.0:7080
    restart: unless-stopped

volumes:
  portunus-data:
    name: portunus-data
EOF
```

3. 启动服务并验证：

```sh
docker compose up -d && docker compose logs -f -t
```

这已经足够完成默认 Server 安装。自动生成的 TLS 材料和 SQLite 状态都放在
`portunus-data` 卷里。

Web UI 在 Server 主机的 `http://127.0.0.1:7080/`。Client 控制面 listener
仍然是 `0.0.0.0:7443`。

**部署在反向代理 / 云主机后面？** enrollment URI 必须嵌入边缘 client
能拨号的**公网** `host:port`。若不配置，未经浏览器请求创建的 enrollment
（例如 Docker operator 的 `enroll-client`）会回落到 `127.0.0.1:7443`，
远程 client 无法连接。通过 `PORTUNUS_ADVERTISED_ENDPOINT=host:port`
（或 Web UI **设置 → 客户端连接地址**）设置通告地址，并确保服务器证书
SAN 覆盖该主机 —— 参见
[通告地址](/zh/docs/features/advertised-endpoint)。

### Compose 文件参考

```yaml
services:
  server:
    image: ghcr.io/zingerlittlebee/portunus-server:latest
    container_name: portunus-server
    ports:
      - "7443:7443"
      - "127.0.0.1:7080:7080"
    volumes:
      - portunus-data:/var/lib/portunus
    command:
      - --data-dir
      - /var/lib/portunus
      - serve
      - --operator-http-listen
      - 0.0.0.0:7080
    restart: unless-stopped

volumes:
  portunus-data:
    name: portunus-data
```

示例默认使用 `:latest`；如果需要完全可复现的生产部署，请固定到具体版本 tag，
例如 `:vX.Y.Z`。

如果需要非默认监听地址或其他覆写，再把可选 `server.toml` 放到
`<data-dir>/server.toml`，例如
`./server.toml:/var/lib/portunus/server.toml:ro`。

操作员命令、Client Compose、备份迁移和生产注意事项见
[Docker 部署](/zh/docs/deployment/docker)。

如果从源码编译：

```sh
git clone https://github.com/ZingerLittleBee/Portunus.git
cd Portunus
cargo build --release -p portunus-server
sudo install -m 0755 target/release/portunus-server /usr/local/bin/
```

如需手动下载二进制归档，请参见下方 [Release 二进制](#release-二进制)。

在 Server 主机上验证：

```sh
portunus-server --version
```

## 安装 Client

最快的路径是一行安装脚本：

```sh
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/scripts/install.sh | bash -s -- client
```

生产环境推荐以容器方式运行 Client——参见下方 Docker 小节。

### 接入并启动

向操作员申请一条接入命令——可通过 Web UI **Clients** 页面；也可在宿主机二进制
部署的服务端执行 `portunus-server enroll-client <name>`；或者对于 Docker 部署的
服务端，执行 `docker compose run --rm operator enroll-client <name>`（参见
[操作员命令](/zh/docs/deployment/docker#操作员命令)）。然后在边缘主机上：

```sh
portunus-client enroll 'portunus://...' --out ./client.bundle.json
portunus-client --bundle ./client.bundle.json
```

参见 [Client 配置 — 接入](/zh/docs/configuration/client#接入enroll)，
了解 bundle 解析顺序、`0600` 权限和默认路径。

### Docker

以宿主用户身份运行一次性接入和长驻容器，确保 `0600` bundle 由同一身份写入与读取：

```sh
# 一次性：把接入命令兑换为宿主文件。
docker run --rm --user "$(id -u):$(id -g)" -v "$PWD:/work" \
  ghcr.io/zingerlittlebee/portunus-client \
  enroll 'portunus://...' --out /work/client.bundle.json

# 长驻转发容器。
docker run -d --name portunus-client --network host \
  --user "$(id -u):$(id -g)" \
  -v "$PWD/client.bundle.json:/etc/portunus/client.bundle.json:ro" \
  ghcr.io/zingerlittlebee/portunus-client
```

<Callout type="warn">
**同机试用？** 当 Server 的 advertised endpoint 是默认的 `127.0.0.1:7443` 时，
一次性接入容器在 Docker bridge 网络下无法连到它（报 `transport error`，不会写出
bundle）——需要给 `enroll` 命令也加上 `--network host`（上面的长驻容器已经用了它），
或设置一个可达的 [advertised endpoint](/zh/docs/features/advertised-endpoint)。
另外请确认 `enroll` 步骤在长驻 `docker run` **之前**确实写出了
`client.bundle.json`：Docker 会把不存在的 bind-mount 源路径创建成一个**目录**，
client 随后会以 `Is a directory` 拒绝它。
</Callout>

Compose 等价写法（`compose.client.yaml`），在 bundle 已存在的情况下：

```yaml
services:
  client:
    image: ghcr.io/zingerlittlebee/portunus-client:latest
    container_name: portunus-client
    network_mode: host
    user: "${HOST_UID}:${HOST_GID}"
    volumes:
      - ./client.bundle.json:/etc/portunus/client.bundle.json:ro
    restart: unless-stopped
```

用以下命令启动：

```sh
HOST_UID=$(id -u) HOST_GID=$(id -g) docker compose -f compose.client.yaml up -d
```

Client 推荐使用 `--network host`，因为转发监听端口是在后续 operator 推送规则时
动态创建的。如果不用 host networking，需要显式发布每个转发监听端口。

参见 [Client 配置](/zh/docs/configuration/client#docker) 了解为何两条命令都要以
宿主用户身份运行，以及 [Docker 部署](/zh/docs/deployment/docker) 了解生产注意事项。

如果从源码编译：

```sh
git clone https://github.com/ZingerLittleBee/Portunus.git
cd Portunus
cargo build --release -p portunus-client
sudo install -m 0755 target/release/portunus-client /usr/local/bin/
```

如需手动下载二进制归档，请参见下方 [Release 二进制](#release-二进制)。

在 Client 主机上验证：

```sh
portunus-client --version
```

## 安装 Standalone

`portunus-standalone` 的所有规则都从一个 TOML 文件读取，无需 Server、无需
enrollment、无需 gRPC。两条安装路径——容器主机用 **Docker**，普通 Linux + systemd
用**一键安装脚本**。

### Docker

```sh
# 1. 下载一份起步配置并编辑：
curl -fsSL -o portunus.toml \
  https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/crates/portunus-standalone/contrib/portunus.example.toml
chmod 0644 portunus.toml
${EDITOR:-vi} portunus.toml

# 2. 运行容器（host 网络，使端口范围规则可直接绑定）：
docker run -d --network host \
  --name portunus-standalone \
  --restart unless-stopped \
  -v "$PWD/portunus.toml:/etc/portunus/standalone.toml:ro" \
  ghcr.io/zingerlittlebee/portunus-standalone:latest
```

容器以 UID 65532（`nonroot`）运行，因此宿主配置文件必须能被该 UID 读取——
`chmod 0644 portunus.toml` 是最简单的修复方式。生产环境请固定版本 tag
（`:vX.Y.Z`）而非 `:latest`。

### 一键安装脚本（二进制 + systemd）

```sh
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/scripts/install.sh | bash -s -- standalone
```

脚本会安装二进制、创建 `portunus` 系统用户、写入起步配置
`/etc/portunus/standalone.toml`，并安装一个加固的 systemd unit。服务**不会**自动
启动——随附配置是模板。先编辑，再启用：

```sh
sudo ${EDITOR:-vi} /etc/portunus/standalone.toml
sudo systemctl enable --now portunus-standalone
```

不启动服务即可校验任意配置（成功时打印 `ok` 并 exit 0）：

```sh
portunus-standalone --check --config /etc/portunus/standalone.toml
```

如果从源码编译：

```sh
git clone https://github.com/ZingerLittleBee/Portunus.git
cd Portunus
cargo build --release -p portunus-standalone
sudo install -m 0755 target/release/portunus-standalone /usr/local/bin/
```

→ 完整的配置 schema、规则示例、CLI 标志、信号、实时 stats TUI 与运维注意事项，
参见 [Standalone 转发器](/zh/docs/configuration/standalone)。

## Release 二进制

推荐的安装方式是 `install.sh`——它会自动解析最新版本，下载正确归档，并校验 SHA-256。

如需手动下载，请访问 GitHub
[releases 页面](https://github.com/ZingerLittleBee/Portunus/releases)，
选择适合你平台的归档。归档命名格式为
`portunus-<version>-<target>.tar.gz`，支持以下四个目标平台：

- `x86_64-unknown-linux-musl`
- `aarch64-unknown-linux-musl`
- `x86_64-apple-darwin`
- `aarch64-apple-darwin`

Linux 归档自 v1.8.0 起是**静态 musl** 二进制：一个归档即可在所有 Linux
发行版（glibc、Alpine/musl、busybox）上运行。每次 release 同时发布对应的
`portunus-<version>-checksums.txt`。

解压并安装：

```sh
# pick the asset for your platform from the releases page, then:
tar -xzf portunus-<version>-<target>.tar.gz
sudo install -m 0755 portunus-<version>-<target>/portunus-server /usr/local/bin/
sudo install -m 0755 portunus-<version>-<target>/portunus-client /usr/local/bin/
```

## systemd

一步完成二进制安装和加固 systemd unit 的安装：

```sh
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/scripts/install.sh | sudo bash -s -- server --systemd
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/scripts/install.sh | sudo bash -s -- client --systemd
```

这会创建服务用户和目录，并安装加固 unit（仅 Linux）。如果使用源码 checkout 安装，
改用 `cd deploy/systemd && sudo ./install.sh server client`（二进制必须已在
`/usr/local/bin`）。

详见 [systemd 部署](/zh/docs/deployment/systemd) 中的 unit 文件解读与加固说明。

## 交互式管理器

无参数运行安装脚本进入向导菜单（管道方式也可用）：

```sh
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/scripts/install.sh | bash
```

提供 **安装 / 卸载 / 升级 / 状态 / 服务 / 配置 / 环境变量**，覆盖
二进制+systemd 与 Docker Compose；server 安装会提示通告地址（持久化到
systemd drop-in 或 compose `.env`）。非交互 flag 不变，供 CI/自动化使用；
`--dry-run` 仍不联网、不写文件。需要 bash 4+。

完整的 verb/flag 列表、部署形态细节、限定 config 键、生命周期示例与安全
语义，参见 [install.sh 管理器参考](/zh/docs/cli/installer)。
