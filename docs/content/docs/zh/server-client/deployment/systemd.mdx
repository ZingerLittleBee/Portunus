`Portunus` 在 `deploy/systemd/` 目录下提供了拿来即用的加固 unit。两份 unit
都以专用系统用户运行，启用 `ProtectSystem=strict`、`@system-service` 系统调用
过滤、受限的地址族集合，以及与负载相称的 `LimitNOFILE`。

## 一键安装

在边缘主机上，一步安装客户端二进制与加固单元：

```sh
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/scripts/install.sh \
  | sudo sh -s -- client
```

该命令会下载并校验 release 二进制、创建 `portunus-client` 服务用户、准备
`/etc/portunus`（mode 0750），并安装加固后的 `portunus-client.service`
（服务默认就会安装 —— `--systemd` 是已废弃的空操作）。server 端用
`… | sudo sh -s -- server`（创建 `portunus-server` 与 `/var/lib/portunus`，
mode 0750）。二进制获取与标志说明统一记录在
[Client 配置](/zh/docs/configuration/client#安装二进制)。

<Callout type="warn">
纯 `client` 安装会立即启用并启动 `portunus-client.service`，但此时服务还没有凭据
bundle，因此会 crash-loop（`Restart=on-failure`，每 5 秒一次）直到完成接入。可以传
`--enroll '<uri>'`（见 first-boot 步骤 6）一步完成安装 + 接入 + 启动，或加
`--no-service` 只安装不启动，再自行放入 `/etc/portunus/client.bundle.json` 并执行
`systemctl enable --now portunus-client`。
</Callout>

如果从源码 checkout 安装，也可以只安装 unit 文件：
`cd deploy/systemd && sudo ./install.sh client`（需要二进制已在 `PATH` 上）。

## server.service

```ini
[Unit]
Description=Portunus control plane
Documentation=https://github.com/ZingerLittleBee/Portunus
After=network-online.target
Wants=network-online.target

[Service]
Type=exec
User=portunus-server
Group=portunus-server
ExecStart=/usr/local/bin/portunus-server --data-dir /var/lib/portunus serve
Restart=on-failure
RestartSec=5
TimeoutStopSec=60
LimitNOFILE=4096

# Hardening
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=true
PrivateTmp=true
PrivateDevices=true
ProtectKernelTunables=true
ProtectKernelModules=true
ProtectControlGroups=true
RestrictNamespaces=true
RestrictRealtime=true
LockPersonality=true
MemoryDenyWriteExecute=true
RestrictAddressFamilies=AF_INET AF_INET6
SystemCallFilter=@system-service
SystemCallErrorNumber=EPERM
ReadWritePaths=/var/lib/portunus

[Install]
WantedBy=multi-user.target
```

`TimeoutStopSec=60` 大于配置的 drain（`shutdown_drain_timeout_secs`，默认
30 秒），确保 server 自身的 drain 在 `SIGKILL` 之前完成。server 不绑定转发端口
（由 client 绑定），因此不需要 `CAP_NET_BIND_SERVICE`；`LimitNOFILE=4096`
足以覆盖 gRPC 流、operator HTTP、metrics 和持久化，可承载数百个 client。
`ReadWritePaths=/var/lib/portunus` 是唯一可写路径：SQLite 状态、自动生成的
TLS 材料，以及可选的 `server.toml`。

## client.service

```ini
[Unit]
Description=Portunus edge client
Documentation=https://github.com/ZingerLittleBee/Portunus
After=network-online.target
Wants=network-online.target

[Service]
Type=exec
User=portunus-client
Group=portunus-client
ExecStart=/usr/local/bin/portunus-client --bundle /etc/portunus/client.bundle.json
Restart=on-failure
RestartSec=5
TimeoutStopSec=60
LimitNOFILE=16384

AmbientCapabilities=CAP_NET_BIND_SERVICE
CapabilityBoundingSet=CAP_NET_BIND_SERVICE

# Hardening
NoNewPrivileges=true
ProtectSystem=strict
ProtectHome=true
PrivateTmp=true
PrivateDevices=true
ProtectKernelTunables=true
ProtectKernelModules=true
ProtectControlGroups=true
RestrictNamespaces=true
RestrictRealtime=true
LockPersonality=true
MemoryDenyWriteExecute=true
RestrictAddressFamilies=AF_INET AF_INET6
SystemCallFilter=@system-service
SystemCallErrorNumber=EPERM
ReadOnlyPaths=/etc/portunus/client.bundle.json

[Install]
WantedBy=multi-user.target
```

client 绑定 operator 推送的规则所配置的 listen 端口，因此带 `CAP_NET_BIND_SERVICE`
以绑定特权端口（小于 1024）。`LimitNOFILE=16384` 覆盖默认 1024 个端口的范围上限
加上在途连接（每个端口占一个 listener fd，每条连接占两个 fd）。在 `server.toml`
中调高 `range_rule_max_ports` 的运维者必须按比例上调该值。bundle 以只读方式挂载。

## 首次启动流程

安装后：

```sh
# 1. 可选：覆写默认值（curl|sh 安装不会留下 checkout —— 从仓库拉取示例；
#    没有该文件时 server 使用内置默认值）
sudo curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/deploy/server.toml.example \
  -o /var/lib/portunus/server.toml
sudo nano /var/lib/portunus/server.toml

# 2. 启用并启动 server
sudo systemctl enable --now portunus-server
sudo systemctl status portunus-server

# 3. 读取 serve 打印的 setup token
sudo journalctl -u portunus-server -n 100 --no-pager \
  | grep 'Portunus onboarding setup token'

# 4. 通过 SSH tunnel 或本机浏览器打开 http://127.0.0.1:7080/，
#    粘贴 setup token，创建第一个 superadmin 密码。

# 5. 创建首个 Client —— Web UI Clients 页面，或离线 CLI。
#    enroll-client 直接打开 state.db，需先停服务
#    （否则会对运行中的 server 报 store_in_use）：
sudo systemctl stop portunus-server
sudo -u portunus-server portunus-server \
  --data-dir /var/lib/portunus enroll-client edge-01
sudo systemctl start portunus-server

# 6. 在边缘主机：一条命令完成安装、接入并启动
curl -fsSL https://raw.githubusercontent.com/ZingerLittleBee/Portunus/main/scripts/install.sh \
  | sh -s -- client --enroll 'portunus://HOST:7443/enroll?pin=sha256:…&code=…'
# --enroll 会放置 bundle（root:portunus-client 0640）并启用服务。
```

setup token 30 分钟后过期，并且在 onboarding 完成前每次 server 启动都会轮换。

## 烟雾测试

上面的 Web onboarding 创建的是基于密码的 superadmin，而不是 API bearer Token。
要探测 operator API，先在 Web UI 的 **Credentials** 页面创建一个 Token，并以
`PORTUNUS_OPERATOR_TOKEN` 导出：

```sh
export PORTUNUS_OPERATOR_TOKEN=<web-ui-生成的-token>
curl -H "Authorization: Bearer $PORTUNUS_OPERATOR_TOKEN" \
     http://127.0.0.1:7080/v1/users/me
```

## 密码恢复

如果最后一个 `superadmin` 忘了 Web 密码，先停服务，在本地重置现有账户，再启动
服务。这里要填真实的超管用户 ID：`bootstrap-superadmin` 创建的是
`_superadmin`，Web onboarding 创建的则是初始化时选择的 ID：

```sh
sudo systemctl stop portunus-server
sudo -u portunus-server portunus-server \
  --data-dir /var/lib/portunus \
  reset-password admin --temporary
sudo systemctl start portunus-server
```

该命令只打印一次临时密码，并要求登录后修改密码。它会直接打开 SQLite store，
因此不要在 `portunus-server.service` 运行时执行。

## 加固备注

- **小于 1024 的端口** 需要 `CAP_NET_BIND_SERVICE`，该能力放在 **client**
  unit 上（client 绑定规则的 listen 端口，server 不绑定）。转发器只绑定规则
  配置过的端口，能力面被严格收敛。
- **`--data-dir` 落在 NFS / tmpfs / ramfs** 时，启动期会以
  `event=startup.unsupported_filesystem` 拒绝。请改用本地文件系统。
- **两个 server 进程同时打开同一 data-dir** 时，第二个会以
  `event=startup.store_in_use`（退出码 1）退出。clustering 不在范围内。

## 备份 cron

```sh
# /etc/cron.daily/portunus-backup
#!/bin/sh
set -eu
DEST=/var/backups/portunus/$(date +%F).db
mkdir -p "$(dirname "$DEST")"
sudo -u portunus-server portunus-server \
  --data-dir /var/lib/portunus \
  backup --out "$DEST"
find /var/backups/portunus -name '*.db' -mtime +30 -delete
```

在线备份走 SQLite Online Backup API —— 无须暂停写入，可在 server 正常服务
期间安全运行。
