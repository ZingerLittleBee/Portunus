`portunus-server` 是控制面二进制。绝大多数子命令都经由 loopback HTTP API
执行，因此需要先设置 `PORTUNUS_OPERATOR_TOKEN`。它的值就是 `server.toml`
里的静态 `operator_token`——用 [`gen-token`](#server-生命周期) 生成一个，
再粘贴进配置：

```sh
export PORTUNUS_OPERATOR_TOKEN=<paste-operator_token-here>
```

v1.1.0 起，Web UI 登录使用本地密码和服务端 session cookie。机器使用静态
`operator_token` 认证；人类使用 Web cookie 登录。

请以 `portunus-server --help`（或 `portunus-server <subcommand> --help`）
为准，下表只是按用途分类的速查。

## Server 生命周期

| 子命令 | 用途 |
| --- | --- |
| `serve` | 启动控制面（gRPC + 操作员 HTTP + metrics） |
| `gen-token` | 输出一段新的 URL-safe-base64 Token |
| `bootstrap-superadmin --name <name>` | 旧的一次性超管 API Token 签发，Bearer 仅打印一次 |
| `onboarding-token` | 对未初始化 store 离线轮换首次 setup token |

```sh
portunus-server --data-dir ./srv serve
```

如果需要非默认的监听地址或其他覆写，可放一个可选的 `./srv/server.toml`。

常规首次设置时，先启动 `serve`，再用运行中的 server 打印的 setup token 完成
Web UI onboarding。`onboarding-token` 会直接打开 store，而 `serve` 启动时又会再次
轮换 token，因此它主要用于离线维护和测试。

## Client

| 子命令 | 用途 |
| --- | --- |
| `enroll-client <name> [--address <host>] [--ttl-secs <N>]` | 打印短期有效的 `portunus-client enroll ...` 命令。`--ttl-secs` 默认 600；`--address` 覆写 Client 拨号的 host。name 是自由格式的显示标签（任意大小写、空格、点、Unicode），允许重名 |
| `revoke <client_name>` | 撤销 Client Bearer，立即断线 |
| `rename-client --client-id <ULID> --name "<display>"` | 修改 Client 的自由格式显示名。按稳定 id 寻址，因此 id——以及以它为键的所有规则、令牌、配额与流量历史——保持不变；不会中断活动会话（v2.0+） |
| `list-clients [--format text\|json]` | 列出所有已签发 Client 的连接状态，含其稳定 `client_id` |

## 规则

| 子命令 | 用途 |
| --- | --- |
| `push-rule <client> <listen_port> <target>` | 推送一条单目标 TCP 规则 |
| `push-rule … --protocol udp` | UDP 变体 |
| `push-rule … --target host:port@<priority>` | 多目标故障切换（可重复） |
| `push-rule … --sni <pattern>` | TLS SNI 路由 |
| `push-rule … --bandwidth-in-bps <N>` 等 | 限流上限（v0.11+） |
| `push-rule <client> <start>-<end> <host>:<start>-<end>` | 端口范围规则 |
| `push-rule … --prefer-ipv6` | DNS 目标 AAAA 优先 |
| `remove-rule <rule_id>` | drain 后移除一条规则 |
| `list-rules` / `list-rules --client <name>` | 列出规则（按 Client 过滤） |
| `rule-stats <id>` | 该规则的聚合计数 |
| `rule-stats <id> --per-port` | 逐端口（端口范围规则） |
| `rule-stats <id> --per-target` | 逐目标（多目标规则） |

## 用户（v0.5+）

| 子命令 | 用途 |
| --- | --- |
| `user-add <id> --display-name <name> [--role admin\|user] --password-stdin` | 添加用户（`--role` 默认 `user`）。必须提供初始密码——通过 `--password-stdin` 管道传入 |
| `user-list` | 列出用户 |
| `user-get <id>` | 查看用户 |
| `user-remove <id>` | 删除用户（级联删除其 Grant 和规则） |
| `reset-password <user_id> [--temporary] [--password-stdin]` | 直接针对 data dir 的本地密码恢复 |

`user-add` 要求提供初始密码；用 `--password-stdin` 从 stdin 第一行管道传入：

```sh
echo 's3cret-pw' | portunus-server user-add alice \
  --display-name Alice --role user --password-stdin
```

`reset-password` 会直接打开 SQLite store。如果 `serve` 正在使用同一个
`--data-dir`，请先把它停掉。

```sh
portunus-server --data-dir /var/lib/portunus \
  reset-password admin --temporary
```

这里要填真实的超管用户 ID：`bootstrap-superadmin` 创建的是 `_superadmin`，
Web onboarding 创建的则是初始化时选定的 ID。

不带 `--temporary` 时，命令会以无回显方式提示输入两次密码。`--password-stdin` 从 stdin
第一行读取新密码，便于自动化。重置密码会撤销该用户的活跃 Web 会话。

## Grant

| 子命令 | 用途 |
| --- | --- |
| `grant-add --user-id <id> --client <name> --listen-port-start <N> --listen-port-end <N> --protocols <tcp,udp>` | 添加 Grant |
| `grant-list` | 列出 Grant |
| `grant-revoke <grant_id>` | 撤销 Grant（级联其依赖规则） |

## 限流（v0.11+）

| 子命令 | 用途 |
| --- | --- |
| `owner-cap set <client> <owner> [--bandwidth-in-bps …]` | 设置每个所有者的信封 |
| `owner-cap get <client> <owner>` | 读信封 |
| `owner-cap list <client>` | 列出该 Client 所有所有者信封 |
| `owner-cap delete <client> <owner>` | 删除信封 |

预校验：`set` 时未提供任何上限会被拒绝，返回退出码 3 +
`validation.rate_limit_no_caps_provided`。

## 存储（v0.8+）

| 子命令 | 用途 |
| --- | --- |
| `backup --out <path>` | 在线 SQLite 备份（兼顾 WAL） |
| `restore --in <path> [--force]` | 恢复备份；自动跑 schema 迁移 |
| `reset --confirm` | 清空 `state.db` + 周边文件 |
| `audit prune --before <RFC3339> [--dry-run]` | 修剪持久审计表 |

## 退出码

| 码 | 含义 |
| --- | --- |
| 0 | 成功 |
| 1 | 通用错误（配置 / IO / `store_in_use` 存储被另一进程占用 等） |
| 2 | 校验错误 |
| 3 | 预校验（如 `validation.rate_limit_no_caps_provided`） |
| 4 | 缺少 `PORTUNUS_OPERATOR_TOKEN` |
| 6 | 激活失败（如 `port_in_use`） |
| 8 | 本地密码恢复目标用户不存在 |
| 78 | schema 比当前二进制还新（试图降级） |

（完整码表参见每个子命令的 `--help`。）

## 常见环境变量

| 变量 | 作用 |
| --- | --- |
| `PORTUNUS_OPERATOR_TOKEN` | 每条操作员子命令的 Bearer |
| `RUST_LOG` | 标准 `tracing-subscriber` env-filter |
