v0.10.0 起支持。两项内容：

1. 每目标可开启的 **PROXY 协议**（v1 文本或 v2 二进制）。
2. SNI 模式 listener 的 **ClientHello peek 时长直方图**。

## PROXY 协议

### 为什么需要它

流量经过转发器中转后，后端看到的 TCP 连接来源是**转发器自己的 IP**，
而不是真正的客户端。原始的 `(源, 目标)` 四元组在这一跳就丢失了，导致：

- **审计 / 日志** —— 访问日志记录的全是转发器 IP，无法把请求溯源到真实访客。
- **基于 IP 的控制** —— 限流、WAF 规则、地理位置识别、黑白名单都以客户端
  IP 为依据，会悄无声息地失效。

PROXY 协议在转发**真实业务数据之前**先把原始地址四元组告诉后端，让后端
能像直连一样恢复出客户端身份，从而解决上述问题。

### 工作方式

后端通过 `proxy_protocol = "v1"` 或 `"v2"` 显式开启时，Client 会在
原始连接的负载之前先发送一段 PROXY 头部。后端就可以看到原始客户端的
`(src_addr, src_port, dst_addr, dst_port)`。

<Callout type="warn">
后端**必须**配置为接收 PROXY 头部。如果给一个不认识 PROXY 协议的后端发送
头部，它会把头部当成业务数据来解析，从而破坏连接。这是一个「两端都要约定
好」的开关 —— 只在后端支持 PROXY 时才逐目标开启。仅 TCP；UDP 路径不受影响。
</Callout>

逐目标开启意味着同一条规则里可以混合 PROXY-aware 与不感知后端：

```sh
portunus-server push-rule edge-01 443 \
  --target backend-a.local:8080@1 --target-proxy-protocol a:v2 \
  --target backend-b.local:8080@2
# backend-a 看到 v2 二进制头部；backend-b 看到旧的字节流
```

（具体 CLI 标记拼写可能略有差异 —— 请以 `portunus-server push-rule --help`
为准。）

## 模式

| 模式 | 描述 |
| --- | --- |
| `v1` | 可读文本头部，例如 `PROXY TCP4 1.2.3.4 5.6.7.8 12345 80\r\n` |
| `v2` | 16 字节二进制头部，遵循 PROXY 协议 v2 规范 |
| 缺省 | 无前缀 —— 旧的字节流 |

选择是 **逐目标** 而非 **逐规则**，所以多目标规则可以混合开关。

## Peek 时长直方图（仅 SNI 模式 listener）

### 有什么作用

在 SNI 路由模式下，listener 必须先「偷看」（peek）入站的 TLS ClientHello
（但不消费它），从中提取 SNI 域名，才能决定这条连接归属哪个后端。这一步
**同步**地串在连接建立路径上：它要等 ClientHello 到齐（可能跨多个 TCP 段），
再解析 TLS 记录。慢客户端、网络抖动或停滞的握手都会让这一步变慢，从而拖累
连接建立的尾延迟。

直方图测量的正是这一步耗时，因此可以用来：

- **观测 SNI 路由的代价** —— 看 p50/p99 的 peek 耗时，判断 SNI 分发是否
  正在成为瓶颈。
- **发现慢客户端 / 攻击迹象** —— 大量长耗时 peek 可能意味着 slowloris 式的
  慢速握手攻击，或网络异常。
- **容量与超时调优** —— 用真实分布数据来设定 peek 超时，而不是拍脑袋。

### 指标

当 listener 处于 [SNI 路由模式](/zh/docs/features/tls-sni-routing) 时，
Portunus 会记录 ClientHello peek + 解析所耗时间：

```
portunus_tls_client_hello_peek_duration_seconds_bucket{...}
portunus_tls_client_hello_peek_duration_seconds_sum
portunus_tls_client_hello_peek_duration_seconds_count
```

桶按 Prometheus 惯例划分，最大有限边界为 **3 秒**；超过 3 秒的观测值会让
`_count` 与 `le="+Inf"` 同步增长，但不会增加 `le="3"`。

旧的纯 TCP listener **不会** 暴露该直方图 —— 那些路径根本不做 peek。
