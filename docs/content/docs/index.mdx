`forward-rs` is a high-performance port-forwarding service written in Rust.
It pairs a central **control-plane server** with one or more **edge clients**
that bind listeners on remote hosts and forward TCP / UDP traffic to
configured upstream targets.

Operators push rules from the server (CLI, HTTP API, or Web UI), and every
edge client picks them up over a TLS-pinned bidirectional gRPC stream.
Per-rule byte / connection / datagram metrics flow back every 5 seconds and
surface via Prometheus and the operator API.

## Why forward-rs?

- **Single binary**, zero runtime dependencies. Bundled SQLite, embedded
  Web UI, vendored TLS — copy the binary, run.
- **TLS + bearer-token** auth on the control plane. Client certificates
  are pinned by SHA-256 fingerprint baked into the issued bundle.
- **Multi-tenant RBAC**. Scope each user to specific clients, port ranges,
  and protocols; per-rule ownership is enforced server-side.
- **L4 byte passthrough**. Never decrypts, terminates, or re-encrypts TLS —
  even when routing by SNI.
- **Operationally honest**. Hot-reload of caps, graceful drain, audit log,
  Prometheus metrics, structured JSON logs, backup/restore tooling.
