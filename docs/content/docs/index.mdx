# Portunus

Portunus forwards TCP and UDP ports on remote hosts. For one machine, run the
standalone binary from a TOML file. For many machines, run a server that pushes
rules to clients and manages permissions, rate limits, metrics, and audit logs.
Business traffic flows through the forwarding host, not through the server, and
Portunus does not decrypt or edit it.

Only need one host? The [standalone forwarder](/en/docs/configuration/standalone)
reads local TOML and uses the same forwarding code, with no server or enrollment
step.

## Performance

On the benchmark host, uncapped TCP forwarding keeps up with `iptables` through
20 Gbit/s. The current performance report is based on v1.6.1 measurements and
includes the host details, commands, raw numbers, and caveats.

| Case | What to expect |
| --- | --- |
| 100 Mbit/s - 20 Gbit/s | Hits the requested rate on the benchmark host. |
| Standalone vs client | Same forwarding code; no measurable throughput difference in the current tests. |
| Rate-limited rules | Keep the same metrics, counters, and audit behavior. |

## What it gives you

| Area | Capabilities |
| --- | --- |
| Forwarding | TCP and UDP rules, port ranges, DNS-name targets, multi-target failover. |
| Routing | Route HTTPS by hostname without decrypting TLS, optional PROXY protocol headers. |
| Management | CLI, HTTP API, embedded Web UI, and live rule updates to connected clients. |
| Access control | Per-user grants by client, protocol, and port range, with server-enforced ownership. |
| Limits | Per-rule and per-owner bandwidth, new-connection, and concurrent-connection caps. |
| Operations | Prometheus metrics, structured logs, audit trail, bundled SQLite, backup and restore. |

## Where it fits

Use `Portunus` when you need to manage forwarding rules across machines, give
users scoped access, watch traffic, or set per-user limits. If you only need one
static local redirect and peak throughput is the only goal, kernel forwarding is
simpler. If you want Portunus forwarding features on one host without a server,
use the [standalone forwarder](/en/docs/configuration/standalone): the same
forwarding code, driven entirely by a TOML file.
