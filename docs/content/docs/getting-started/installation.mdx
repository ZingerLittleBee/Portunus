`Portunus` ships as two single binaries — `portunus-server` (control plane)
and `portunus-client` (edge). Pick the option that matches your environment.

## Prerequisites

- **Rust 1.88+ stable** (only when building from source). `protoc` is
  vendored via `prost-build` — no system install required.
- **Two reachable hosts** for a real deployment. For local trials,
  one host with two terminals is enough.
- **Linux or macOS**. Linux is the supported production target.

## Option 1 — Build from source (recommended)

```sh
git clone https://github.com/ZingerLittleBee/Portunus.git
cd Portunus
cargo build --release -p portunus-server -p portunus-client
```

You now have:

```
target/release/portunus-server
target/release/portunus-client
```

Copy them to `/usr/local/bin/` (or anywhere on `PATH`) and you are done.

<Callout type="info">
  The release profile uses `lto = "thin"`, `codegen-units = 1`, and
  `strip = "symbols"`. A clean build takes ~3-5 minutes on a modern laptop.
</Callout>

## Option 2 — Docker

The repo ships multi-stage Dockerfiles that produce
`distroless/cc:nonroot` images.

```sh
# Server
docker build -t portunus-server:latest -f deploy/docker/Dockerfile.server .

# Client
docker build -t portunus-client:latest -f deploy/docker/Dockerfile.client .
```

A local-only `docker-compose.yml` lives at `deploy/docker/docker-compose.yml`
for kicking the tires.

## Option 3 — systemd (production)

The repo ships hardened systemd units under `deploy/systemd/` with
`User=`, `ProtectSystem=`, `CapabilityBoundingSet=`, and an `install.sh`
that creates the service users and lays out the FHS directories.

```sh
cd deploy/systemd
sudo ./install.sh
# Creates:
#   /etc/portunus/        admin-edited config + TLS material
#   /var/lib/portunus/    daemon-managed state.db
#   portunus system user
```

See [Deploying with systemd](/docs/deployment/systemd) for the full unit
breakdown and hardening notes.

## Verify the install

```sh
portunus-server --version
portunus-client --version
```

They should print `portunus-server <version>` and `portunus-client <version>`.
