This walkthrough mirrors `quickstart.md` from the v0.1 spec — it doubles as
the script the `forward-e2e` integration tests exercise programmatically.

## 0. Prerequisites

You need [`forward-server` and `forward-client` installed](/docs/getting-started/installation)
and on `PATH`, plus two terminals (or two hosts).

## 1. Start the server (Host A)

```sh
mkdir -p ./srv
forward-server --config-dir ./srv \
  --control-listen 0.0.0.0:7443 \
  --operator-http-listen 127.0.0.1:7080 \
  --metrics-listen 127.0.0.1:7081 \
  serve
```

On first launch the server creates `./srv/server.crt`, `./srv/server.key`,
and an empty `./srv/tokens.json`. Logs are JSON lines:

```json
{"event":"tls.cert_generated","fingerprint_sha256":"…"}
{"event":"control.listening","addr":"0.0.0.0:7443"}
{"event":"operator_http.listening","addr":"127.0.0.1:7080"}
```

## 2. Bootstrap the superadmin operator

The operator API is bearer-authed (v0.5.0+). Mint a superadmin once:

```sh
forward-server --config-dir ./srv bootstrap-superadmin --name ops
# → superadmin user_id=_superadmin token=<bearer>
```

<Callout type="warn">
  The bearer is printed **exactly once**. Capture it now.
</Callout>

```sh
export FORWARD_OPERATOR_TOKEN=<paste-token-here>
```

Every operator subcommand below reads `FORWARD_OPERATOR_TOKEN` from env.

## 3. Provision a forwarding client

```sh
forward-server --config-dir ./srv \
  provision-client edge-01 --out ./srv/edge-01.bundle.json
```

The bundle (mode 0600) contains the server endpoint, pinned cert SHA-256,
and the client's bearer token. Transfer it to Host B over a trusted channel.

## 4. Start the client (Host B)

```sh
forward-client --bundle ./edge-01.bundle.json
```

Within a few seconds the server logs `client.connected`. Verify:

```sh
forward-server --config-dir ./srv list-clients
# NAME      CONNECTED   REMOTE              CONNECTED-AT
# edge-01   yes         203.0.113.5:51234   2026-05-06T12:01:30Z
```

## 5. Push a forwarding rule

```sh
# 18080 on edge-01 → 10.0.0.5:8080
forward-server push-rule edge-01 18080 10.0.0.5:8080
# → rule_id=1
```

The rule transitions `Pending → Active` with a 1-second ack budget.
The client now binds `0.0.0.0:18080` and forwards every accepted TCP
connection to `10.0.0.5:8080`.

## 6. Stream traffic through the rule

```sh
# Send 100 MB (assuming nc -lk 8080 > /tmp/out.bin on the target)
dd if=/dev/urandom of=/tmp/in.bin bs=1M count=100
nc <host-B> 18080 < /tmp/in.bin

sha256sum /tmp/in.bin /tmp/out.bin   # hashes must match
```

## 7. Observe

```sh
# Aggregate counters via the operator CLI
forward-server rule-stats 1
# → rule_id=1 bytes_in=104857600 bytes_out=0 active_connections=0

# Same numbers via Prometheus
curl -s http://127.0.0.1:7081/metrics | grep '^forward_rule'
```

## 8. Tear down

```sh
forward-server remove-rule 1
# Rule drains, in-flight connections finish within shutdown_drain_timeout_secs.
```

`SIGTERM` either binary; both drain in-flight connections (default 30 s)
then exit `0`.
